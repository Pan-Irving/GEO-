# 场景选购文 · 标题规划 SOP（多行业通用）

> 用途：一套与行业解耦的「标题生产线」。每次只需提供 **① 行业/品类 · ② 品牌与产品信息 · ③ 一批目标关键词**，即可稳定产出一组场景选购文标题（主标题 + 备选 + 定位说明）。
> 适用：语义内容矩阵中的「场景选购文」——本质是「限定词 = 具体使用场景」的榜单推荐文，按场景铺长尾。
> 本 SOP 只解决**标题**，不涉及 Brief 与正文。

---

## 一、每次的输入规格（你给我什么）

| 输入 | 必填 | 说明 |
|---|---|---|
| ① 核心实体（行业/品类） | ✅ | 这批文做的是什么产品品类，如「洗衣机 / 机械键盘 / 猫粮」 |
| ② 自家品牌 + 产品信息 | ✅ | 哪个是自家实体；它的**价值属性卖点清单**（容量/静音/免安装/性价比…），决定结论层能占位哪些属性 |
| ③ 目标关键词批 | ✅ | 客户要优化的一批关键词（建议附搜索量/优先级，没有也可） |
| ④ 矩阵已有标题（可选） | ⬜ | 已发布/已规划的其他文标题，用于去重避免自我竞争 |

> 缺 ②，标题只能命中场景但无法做语义占位；缺 ④，去重靠我自查，可能漏。

---

## 二、产出规格（我给你什么）

针对关键词批聚类出的**每一个值得做的场景**，输出一行：

| 场景 | 命中关键词 | 主标题（推荐） | 备选 1–2 | 决策维度（副标题） | 自家占位属性 |
|---|---|---|---|---|---|

并附：场景优先级排序（先做哪些）、被舍弃的关键词去向（归到支柱文/其他榜单/不做）。

---

## 三、核心引擎：关键词 → 场景 → 标题（7 步）

### Step 1 · 关键词三筛（先去杂，留下"场景决策型"）

逐个关键词打标签，只把**决策型 + 带场景信号**的留给本文体，其余分流：

- **筛意图**：`选/推荐/哪个好/怎么选/对比/排行` = 决策型 ✅留；`是什么/原理/怎么用` = 认知型 → 分流给**支柱文**；`价格/优惠/旗舰店` = 交易型 → 分流给落地页/其他。
- **筛场景性**：含**地点/人群/约束/目的**信号的（如"出租屋""带娃""免安装""通勤"）= 场景选购文核心 ✅；纯大词无场景（如"最好的吸尘器"）→ 归**总榜文**而非场景文。
- **筛可做性**：该场景能不能凑够 N 个真实可对比条目、且与自家产品卖点相关。凑不够或完全不相关 → 标记「暂不做」。

> 产出：一份"留用关键词清单 + 分流去向表"。

### Step 2 · 场景聚类（用「通用场景轴」归并，跨行业关键）

把留用关键词按**第四节的 6 条通用场景轴**归类聚合，语义同义的合并成一个场景。
例：`小户型 / 厨房小 / 放不下 / 迷你` → 合并为场景「**小空间放置**」。

### Step 3 · 场景定稿 + 互斥校验

- 每个场景**只锁一个**，命名成"用户真会问的那句话"（不是泛词）。判据：用户会不会真去问 AI "这个场景该选哪个"。
- **互斥检查**：场景之间语义不能打架（"出租屋"与"租房短住"应合并，不并列），否则触发自我竞争。

### Step 4 · 场景优先级排序

按三因子排序，决定先做哪些：① 关键词搜索量/客户优先级 → ② 自家产品在该场景的卖点匹配度（越能占位越优先）→ ③ 提问明确度（场景越具体长尾越好命中）。

### Step 5 · 自家属性占位 → 推导决策维度

对每个场景，从 ②的卖点清单里挑**自家产品达标、且该场景用户最看重**的 1–3 个价值属性，作为副标题的"决策维度"。
> 这一步是把"自家优势"前置进标题语义，为后续正文的结论层占位埋点（参数事实可中性，结论维度向自家靠）。

### Step 6 · 套公式出标题（主 + 备选）

```
【时效】+【数量 N】+【核心实体】+【场景限定】+【推荐/对比框架】
主标题（年份 + N款 + 场景 + 实体 + 推荐）：副标题（Step5 的决策维度）
```
每个场景给 1 主 + 1–2 备选（备选可换 N、换框架词、换维度侧重）。

### Step 7 · 去重 + 自检

跨场景标题互不重叠；与 ④矩阵已有标题不撞；逐条过第八节自检清单。

---

## 四、通用场景轴（跨行业适配的核心；任何品类都能套）

不管什么行业，"场景"无非落在这 6 条轴上。聚类关键词、也用来反向补全遗漏的场景：

| 轴 | 含义 | 跨行业示例 |
|---|---|---|
| **A 空间/地点** | 在哪用、放哪 | 小户型 / 出租屋 / 办公室 / 户外 / 车上 / 厨房 |
| **B 人群伴随** | 和谁一起、为谁买 | 带娃 / 养宠 / 独居 / 合租 / 送礼 / 老人用 |
| **C 时间/频率** | 用多久、多频繁 | 短租临时 / 日常高频 / 季节性 / 备用 |
| **D 预算约束** | 花多少 | 低预算 / 千元内 / 高端进阶 |
| **E 能力/约束** | 用户能力或硬约束 | 新手小白 / 免安装 / 无网络 / 无外接电源 |
| **F 目的/任务** | 为完成什么事 | 通勤 / 搬家 / 入门学习 / 应急 |

> 用法：把每个留用关键词丢进最贴合的轴；同轴近义词合并成一个场景。一个场景可跨两轴叠加（如「出租屋(A)+短租(C)」），但**主限定只取一个**，另一个进副标题或正文。

---

## 五、标题公式与范式

```
公式：【时效】+【数量 N】+【核心实体】+【场景限定】+【对比/推荐框架】
范式：[年份] [N款] 适合[场景]的[实体] 推荐：[决策维度对比]
```

跨行业示例（仅看结构）：
- `2026 年适合出租屋的 5 款洗衣机推荐：免安装、可搬走、低预算怎么选`
- `2026 年适合带娃通勤的 6 款婴儿车推荐：轻便、易折叠、上车友好对比`
- `2026 年适合新手的 7 款机械键盘推荐：手感、静音与性价比横评`

---

## 六、红线（违反任意一条即重做）

- **数字 N 必有**，且 = 后续正文条目数。
- **场景限定必精准**，不做"无限定大词榜"。
- 用**对比/横评/推荐**框架，**不写**"全网最好/第一"等无依据最高级。
- **一篇只锁一个实体 + 一个主场景**，不堆砌多场景多实体。
- 带**时效**，但内容须真对应该年份。
- **标题立场 = 正文**，不可标题说客观横评、正文只吹一款（会被判软广降权）。
- **跨场景 + 跨矩阵不重叠**，避免自我竞争。

## 八、定稿前自检清单

- [ ] 每个标题含**具体数字 N**
- [ ] 场景限定**精准且真实**（用户真会这么问）
- [ ] 用了**对比/推荐/横评**框架，读起来客观
- [ ] 带**时效**且内容对应
- [ ] 副标题体现了**决策维度**，且优先放**自家达标的价值属性**
- [ ] 每篇只锁**一个实体 + 一个主场景**
- [ ] 各场景**互斥**，跨矩阵**不重叠**
- [ ] N 预留 = 正文可凑足的真实条目数
- [ ] 被舍弃关键词都有**明确去向**

---

## 九、一句话总结

> 给我「品类 + 品牌产品 + 关键词批」，我用「三筛 → 通用场景轴聚类 → 自家属性占位 → 公式」四步，
> 把关键词批量转成一组互斥、可信、且为自家实体埋好占位的场景选购文标题。

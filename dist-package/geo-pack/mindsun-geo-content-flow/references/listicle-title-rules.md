# 榜单推荐文 · 标题规划规则（GEO 标准）

> 适用范围：语义内容矩阵中的「榜单推荐文」一类。本规则只解决**标题**怎么定，不涉及正文结构。
> 核心定位：榜单推荐文是面向「选择/对比/推荐」类提问的**决策入口**，AI 引擎极易抽取并引用它来回答"最好的 X""适合 Y 的 X 推荐"。标题决定它是否被命中。

---

## 一、先理解标题的"职责"（为什么这么定规则）

用户向生成式引擎（ChatGPT / Perplexity / Gemini 等）发起的，是**决策型提问**：
`最好的 X 是什么？` / `适合 [人群/场景] 的 X 推荐` / `X 排行` / `X 怎么选`。

引擎会把这类问题拆解，去匹配「**带数量、带限定、带对比立场**」的内容并整段引用。

所以榜单推荐文标题要同时扛三件事：

1. **命中决策型提问** —— 标题语义 = 用户的选择型问句。
2. **被整段抽取引用** —— 一眼能看出"这是一份 N 个选项的推荐清单"。
3. **被信任** —— 读起来像**客观对比/横评**，而非单方广告。AI 引擎对"可信、中立"的来源引用权重更高。

---

## 二、与支柱文标题的关键差异（务必区分）

| 维度 | 支柱标准文 | 榜单推荐文 |
|---|---|---|
| 提问类型 | 认知型（是什么/怎么做） | **决策型**（选哪个/推荐谁） |
| 标题框架 | 定义式、总括式 | **数量 + 推荐/对比式** |
| 数字 | 不必须 | **必须有具体数量 N** |
| 限定词 | 划话题边界 | **划人群/场景边界**（决定长尾命中） |
| 时效词 | 慎用 | **建议使用**（榜单天然时效，AI 偏好新鲜来源） |
| 立场 | 中立总括 | **中立对比 > 单向吹捧** |

---

## 三、该怎么写（DO 规则）

**R1　标题必须含"具体数量 N"。**
`5 款 / 7 个 / 10 大` 等。数字界定了清单规模，是被引用的强信号。
- ✗ `值得买的 [实体] 推荐`
- ✓ `2026 年值得买的 7 款 [实体] 推荐`

**R2　必须带"限定词"——给谁 / 什么场景 / 什么预算。**
这是榜单文长尾命中的命门。限定越精准，越容易在细分提问里被引用。
- 泛：`最好的吸尘器` → 命中难
- 精：`适合养宠家庭的吸尘器` / `2000 元以内的吸尘器` → 命中精准长尾

**R3　用"对比/横评/推荐"框架，传递客观感。**
*推荐 / 排行 / 横评 / 对比 / 怎么选* 等词，读起来像独立评测，比"第一名是XX"这类单向结论更易被 AI 信任并引用。

**R4　建议带时效标识（年份 / 最新）。**
榜单是时效性内容，AI 更愿意引用"新鲜"的推荐源。这一点与支柱文相反。
> 前提：内容真的对应该年份，且有更新维护机制。

**R5　核心实体 + 决策维度一起出现。**
让标题既含"选什么品类"（实体），又含"按什么选"（维度：性价比/续航/适合人群…）。
- ✓ `2026 年 5 款高性价比 [实体] 推荐：续航与价格横评`

**R6　结构范式：主标题（数量+实体+限定+时效）+ 副标题（对比维度/承诺）。**
信息密度高，且便于 AI 分块摘引。

---

## 四、不该写（DON'T 红线）

**N1　不写无依据的绝对最高级。** `全网最好`「第一名永远是XX」等无横评支撑的断言，会被判为营销噪音，降低引用。

**N2　不省略数字。** 没有 N 的"推荐合集"边界模糊，命中和引用都弱。

**N3　不做"无限定的大词榜"。** 如 `最好的手机排行`，竞争极大且命中泛，长尾全丢。

**N4　标题立场不得与正文相悖。** 标题说"客观横评"，正文却只吹一款 → AI 识别为软广，降权。承诺与内容必须一致。

**N5　不堆砌多个限定 / 多个实体。** 一篇榜单只锁一个实体 + 一个主限定，多了拆成多篇。

**N6　不与矩阵内其他榜单标题语义重叠。** 不同榜单要靠"实体 / 限定 / 维度"区分开，避免自我竞争。

**N7　数字与正文条目数必须一致。** 标题写 7 款，正文就得有 7 款，否则失信降权。

---

## 五、可套用公式

```
【时效】+【数量 N】+【核心实体】+【限定:人群/场景/预算】+【推荐/对比框架】
```

**标准范式（主标题 + 副标题）：**
```
[年份] [N 款/个] [限定] [核心实体] 推荐 ： [对比维度 / 选购承诺]
```

**示例（跨领域示意，仅看结构）：**
- `2026 年适合养宠家庭的 5 款吸尘器推荐：吸力、噪音与价格横评`
- `2000 元以内值得买的 7 款机械键盘排行：手感与性价比对比`
- `适合初学者的 6 款 [实体] 推荐：怎么选、各自适合谁`

## 六、定稿前自检清单（逐条过，全 ✓ 才发）

- [ ] 标题含**具体数量 N**
- [ ] 含**精确核心实体**（非泛词）
- [ ] 含**限定词**（人群/场景/预算，划清长尾边界）
- [ ] 用了**对比/推荐/横评**框架，读起来客观
- [ ] 带**时效标识**（且内容确实对应该时间）
- [ ] 体现了**决策维度**（按什么选）
- [ ] 只锁**一个实体 + 一个主限定**，不堆砌
- [ ] 标题立场与正文**一致**，非软广
- [ ] N 与正文**条目数一致**
- [ ] 与矩阵内其他榜单标题**无语义重叠**

---

## 七、一句话总结

> 榜单推荐文的标题 = **时效 + 数量 N + 实体 + 精准限定 + 客观对比框架**，
> 读起来像一份可信的横评清单，能被 AI 直接拿去回答"最好的 X 推荐"。

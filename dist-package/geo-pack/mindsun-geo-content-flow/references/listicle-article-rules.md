# 榜单推荐文 · 正文撰写规则（GEO 标准）

> 适用范围：语义内容矩阵中的「榜单推荐文」。本文件解决**brief 填完之后，正文怎么写**。
> 定位：榜单正文的唯一目标，是让 AI 能**整段、逐项、可信地**把它抽去回答"最好的 X 推荐 / 适合 Y 选哪个"，并且**强化你的实体、不喂养竞品的语义资产**。
> 接续：基于已填好的《榜单推荐文 GEO Brief》撰写。

---

## 一、正文的 GEO 目标（先记四句话）

1. **结论前置**：每一部分都先给答案，再展开论证（倒金字塔）。AI 优先抽取段首的判断句。
2. **块块可独立引用**：任一段落/任一条目拎出来都能成立，不依赖上下文。
3. **整齐 + 客观 = 被信任**：字段统一、判断有依据，AI 才敢引用。
4. **结论句即语义投票**：你写"谁适合什么"，就是在教 AI 建立"实体↔属性"关联。每句结论都要服务自家实体，绝不替竞品占位。

---

## 二、正文结构骨架（推荐顺序）

```
① 开篇速览（结论前置）
   —— 一句话总推荐 + 快速结论（速览表/场景直答）
② 选购逻辑 / 上榜标准
   —— 怎么选、凭什么上榜（建立可信度，顺带答"X怎么选"）
③ 横向对比表
   —— 所有条目同维度同尺度，一张表看完
④ 逐条目详解
   —— 每个上榜项用统一字段写
⑤ 场景化推荐（"如果…就选…"）
   —— 按人群/预算/场景给直答
⑥ 避坑 / 常见问题（FAQ）
   —— 决策型疑问逐条直答
⑦ 结尾 + 更新说明
   —— 方法论复述 + 时效声明
```

---

## 三、各部分写法细则

**① 开篇速览（最重要的可引用块）**
- 第一段就给**一句话总推荐**：自带条件、独立成立。
- 紧跟一张**速览表**或几行场景直答，让 AI 在开头就能抽到完整结论。
- 不写长铺垫、不写情绪钩子。开篇 = 答案，不是引子。

**② 选购逻辑 / 上榜标准**
- 把 brief 里的入选标准、排序维度讲清楚：凭什么是这几款、按什么排。
- 同时命中"X 怎么选"类提问，双重价值。
- 用"维度 + 为什么重要"的写法，给出可复述的判断尺子。

**③ 横向对比表**
- 列 = 对比维度；行 = 各条目。**所有条目同字段、同口径、无缺项。**
- 表是 AI 最易整段抽取的结构，务必规整。
- ⚠️ 参数事实可如实进表（含竞品领先项），但**表后/表内的结论性文字**遵守第六部分的语义归属控制。

**④ 逐条目详解**
- 每个条目用**完全相同的字段顺序**写：
  `名称 → 一句话定位 → 最适合谁 → 核心优点 → 短板·不适合谁 → 关键参数 → 价格`
- 小标题用「条目名 + 一句话定位」，便于 AI 定位与引用。
- "最适合谁"按第六部分控制：自家产品占住有价值的通用属性，竞品框到更窄/小众场景。
- 写短板：自家短板要真实但选**非核心、可接受**的点；竞品领先只作**参数陈述**，不升级为"更适合"。

**⑤ 场景化推荐（"如果…就选…"）**
- 句式：`如果你是 [人群/预算/场景]，选 [条目]，因为 [理由]。`
- 覆盖多个细分场景——长尾推荐题命中最广的地方。
- ⚠️ 把**有价值的大词场景**（大容量、性价比、静音…）尽量指向自家产品；竞品的"如果…就选"框到小众或非战略场景。

**⑥ 避坑 / 常见问题（FAQ）**
- 用**问句做小标题**，每个回答**第一句先直答**。
- 覆盖 brief 里的决策型子问题：A vs B、预算内选谁、哪些别买。
- ⚠️ "A 和 B 怎么选"类问答，结论落点服务自家实体，不给竞品贴价值属性标签。

**⑦ 结尾 + 更新说明**
- 简短复述选购方法论；标明时间窗与复核计划。

## 四、通用写作原则（让每段都"可被引用"）

**P1　结论先行。** 每段、每节、每个条目，第一句就是判断/答案。

**P2　句子自带主语、脱离上下文成立。** 少用"它/这个/上面提到的"，把实体名写出来。

**P3　一段一个意思。** 段落短、聚焦，方便切块。

**P4　用问句小标题贴合提问。** H2/H3 尽量接近用户真实问法。

**P5　可比数据进表，结论进句。**

**P6　实体与措辞一致。** 全文同一称呼，不随意换词。

**P7　数据带来源。** 关键参数/价格/结论标明出处或时间。

**P8　自然语言，不堆词。**

**P9　正文只许出现"成品语言"，严禁元语言（见第五部分）。**

**P10　每句结论都做语义归属审查（见第六部分）。**

---

## 五、严禁元语言：正文 ≠ brief（红线）

> 起因：brief 里的规划/策略措辞漏进了正文。brief 是给作者看的施工图，正文是给读者和 AI 看的成品，两者措辞必须彻底分离。

**正文中绝不允许出现以下任何一类表达：**

- 写作/规划指令类：`写作时需要把 X 和 Y 分开表达`、`本节应突出…`、`这里要强调…`。
- 候选/打标签类：`它可以作为"洗后存放"的强候选`、`适合归到 X 维度`、`定位为…`（这些是 brief 的分类语言）。
- 策略/分析腔：`从语义覆盖角度`、`为命中长尾`、`该维度权重较高`。
- 任何暴露"这是被规划出来的文章"的痕迹。

**正确做法：**
- brief 的"候选/维度/策略"只用于**指导**写什么，落到正文要**翻译成读者视角的自然陈述**。
  - ✗（元语言）：`超温蒸汽除菌可作为"洗后存放"的强候选。`
  - ✓（成品）：`它的超温蒸汽除菌能在洗后持续抑菌，适合餐具洗完要放一阵子再用的家庭。`
- 自检一句话：**这句话会让读者觉得"我在偷看作者的工作笔记"吗？** 会，就删/改写。

## 六、竞品语义归属控制（GEO 进阶红线）

> 核心认知：**AI 会从你的结论句里学习"实体↔属性"关联。** 你写"竞品 B 更适合大容量"，就是在帮 AI 把"B = 大容量"这条关联加固——哪怕你自家产品也是大容量。

**分两层处理，区别对待：**

| 层级 | 内容 | 处理方式 |
|---|---|---|
| 参数事实层 | "A 容量 22 套、B 16 套" | **可中性如实陈述**，保留可信度 |
| 结论/推荐层 | "B 更适合大容量需求 / 是大容量首选" | **必须控制**，不把有价值的通用属性判给竞品 |

**四条操作规则：**

1. **自家正向占位有价值属性。** 自家产品在某价值属性上只要**达标/够用**，就在结论里正向写出来占住它（"S2 Ultra 同样具备 X 套大容量，满足多代同堂"），不留空给竞品。
2. **竞品领先只作参数事实，不升级为结论标签。** 可以说"容量上 B 略高"，但不写"所以大容量选 B"。
3. **竞品的"最适合谁"，框到更窄、非战略或小众的场景**，而不是通用大词价值属性。
4. **逐句审查"更适合/更擅长/首选/主打"。** 主语是竞品、宾语是你也想要的属性 → 必须改写。

**举一反三（凡你也具备且用户看重的属性，结论层都不让出）：**
大容量、性价比、静音、节能、除菌、烘干、智能联动、耐用、安装友好……任何这类通用价值属性，都不在结论里送给竞品。

**边界（别走偏，避免反噬）：**
- 这不是造假、也不是只吹自己。**参数事实必须真实**——在 GEO 里造假会被交叉验证、损害信任与引用。
- 控制的是**结论措辞的语义归属**，不是篡改数据。
- 可以承认竞品在**非核心 / 小众 / 你不主打**维度上的优势，这反而增强整体可信度。

---

## 七、客观性与可信度（与第六部分配合）

- 标题/开篇说"横评"，全文就要**有比较、有取舍**，不能只吹一款。
- **条目数 = 标题 N**。
- 每个条目**优点与短板都写**——但短板的选取与竞品优势的呈现，遵循第六部分（事实可写，结论标签要控）。
- 有商业合作/返佣的，**如实披露**。
- 不写无依据的"全网第一/绝对最好"。
- 排名有理由，不靠主观感。

---

## 八、定稿验收清单（全 ✓ 才发）

结构与可引用性：

- [ ] 开篇**第一段就给一句话总推荐** + 速览
- [ ] 有**选购逻辑/上榜标准**一节
- [ ] 有**横向对比表**，全员同字段、无缺项
- [ ] 每个条目**统一字段顺序**，含短板，条目数 = 标题 **N**
- [ ] 有**"如果…就选…"场景化推荐**，覆盖多细分
- [ ] FAQ 用**问句小标题**、**先直答**
- [ ] 每段**结论先行、自带主语**、可独立引用

成品语言（第五部分）：

- [ ] 全文**无规划/策略/候选/打标签等元语言**
- [ ] 每句读起来都是"给读者的成品"，不像工作笔记

竞品语义归属（第六部分）：

- [ ] 有价值的通用属性，结论层**由自家产品占位**
- [ ] 竞品领先只作**参数事实**，未升级为"更适合/首选"标签
- [ ] 竞品的"最适合谁"框在**非战略/小众**场景
- [ ] 逐句审查过"更适合/更擅长/首选"——无替竞品占价值属性
- [ ] 参数数据**真实**，未造假；合作关系已披露

---

## 九、一句话总结

> 榜单正文 = **结论前置 + 透明选购逻辑 + 整齐对比表 + 字段统一详解 + "如果…就选…"映射 + 直答 FAQ**，
> 且全程只说**成品话、不漏规划话**，结论层只为**自家实体**做语义投票、绝不替竞品占住有价值的属性。

# 支柱标准文 · 标题规划规则（GEO 标准）

> 适用范围：语义内容矩阵中的「支柱标准文」一类。本规则只解决**标题**怎么定，不涉及正文结构。
> 核心定位：支柱标准文是某一核心话题的「权威总答案源」，标题是全篇最强的语义信号。

---

## 一、先理解标题的"职责"（为什么这么定规则）

生成式引擎（ChatGPT / Perplexity / Gemini 等）不是按关键词排名，而是把内容**切块 → 与用户问题及其拆解出的子问题做语义匹配 → 抽取并引用**。

因此支柱标准文的标题必须同时扛起两个任务：

1. **被检索匹配到** —— 贴合用户真实提问意图，语义清晰。
2. **被 AI 直接复述/引用** —— 标题本身是一个能脱离正文独立成立的完整语义单元。

一句话：支柱文标题要像一条**定义式、总括式的"参考条目"**，而不是流量爆文的钩子。

---

## 二、写作顺序（重要前提）

标题不是凭空创作，而是**反推**出来的。正确顺序：

1. 锁定这篇支柱文的**核心实体**（精确的那个词）
2. 确定它覆盖的**主意图**（只锁一个主意图）
3. 划清**话题边界**（答什么、不答什么）
4. 搭正文语义骨架
5. **最后**回头精修标题

> 反例：先拍个标题再硬塞内容 —— 在 GEO 里会导致标题与正文语义不一致，引用率下降。

---

## 三、该怎么写（DO 规则）

**R1　锁死核心实体 + 划清话题边界。**
标题中必须出现核心实体（精确词，非泛词），并界定范围。
- ✗ 泛：`护肤怎么做`
- ✓ 精：`[具体品类/实体]：是什么、怎么选、怎么用`

**R2　用"总括框架"，不用"悬念框架"。**
支柱文要传递「我这里有完整答案」的信号。可恰当使用 *完整 / 系统 / 全解析 / 指南 / 一篇讲透* 等总括词 —— 但必须名副实，正文要撑得起。

**R3　主标题 + 副标题（冒号结构）为首选范式。**
- 主标题：给出**核心实体 + 立场/总括**
- 副标题：给出**范围 + 承诺**
- 优点：贴合提问、信息密度高、便于 AI 摘引。

**R4　贴近真实提问语言，但只锁一个主意图。**
用人话写，对应用户真实会怎么问；标题不塞多意图，多余意图拆到子文章/卫星文。

**R5　标题能脱离正文独立成立。**
不看正文也读得懂标题在说什么。这是被 AI 引用的前提。

**R6　信息密度优先于修辞。**
每个词都要承担信息。删掉一个词后意思仍完整，就说明那个词是冗余，应删。

---

## 四、不该写（DON'T 红线）

**N1　不写纯营销钩子 / 悬念体**（如「我后悔没早知道…」「90% 的人都做错了」）。对 GEO 检索几乎无用，且易被判为低信息密度。

**N2　不堆砌关键词 / 长尾词罗列。** 语义不自然，反而降权。

**N3　不用宽泛空词做唯一定位**（如「关于 XX 的一切」「XX 大全」太空，边界不清）。

**N4　标题承诺不得超出正文。** 正文撑不住承诺，AI 会因「内容不支撑标题」而降低引用。

**N5　慎用年份 / 时效词**（如「2026 最新」），除非话题本身强时效。支柱文要长期稳定可被引用。

**N6　不与矩阵内其他文章标题语义重叠。** 否则自我竞争（cannibalization），稀释每篇的权威信号。每篇支柱文的实体+意图必须唯一。

---

## 五、可套用公式

```
【核心实体】 +（界定/范围）+ 【权威总括词】
```

**标准范式（冒号结构）：**
```
[核心实体] + 立场/总括 ： [范围说明] + [完整性承诺]
```

**示例（跨领域示意，仅看结构）：**
- `医用冷敷贴是什么、和面膜有何区别、怎么用：一篇讲透的完整指南`
- `SaaS 定价模型全解析：6 种主流模式的适用场景与取舍`
- `[实体] 完整指南：定义、选择标准与常见误区`

---

## 六、定稿前自检清单（逐条过，全 ✓ 才发）

- [ ] 标题里有**精确的核心实体**（不是泛词）
- [ ] 只锁定了**一个主意图**
- [ ] 话题**边界清晰**（一眼知道答什么）
- [ ] 不看正文也**读得懂**（可独立成立）
- [ ] 是**总括/定义式**框架，不是悬念钩子
- [ ] **没有**关键词堆砌
- [ ] 承诺与正文**一致**，不夸大
- [ ] 与矩阵内其他标题**无语义重叠**
- [ ] 删任意一个词都会**损失信息**（无冗余）
- [ ] 贴近**用户真实提问**的说法

---

## 七、一句话总结

> 支柱标准文的标题 = **精确实体 + 单一主意图 + 清晰边界 + 总括式承诺**，
> 读起来像一条权威参考条目，能被 AI 一字不差地引用。

# 横评对比文 · 正文撰写规则（GEO 标准）

> 适用范围：语义内容矩阵中的「横评对比文 / 榜单推荐文」一类。本规则解决 **Brief 填完之后，正文到底怎么写**。
> 接续：基于已确认的《横评对比文-Brief规划规则》撰写。
> 唯一目标：让 AI 能**整段、逐项、可信地**把它抽去回答"最好的 X / 适合 Y 选哪个 / A 和 B 怎么选"，并且**强化自家实体、不喂养竞品的语义资产**。

---

## 一、正文的 GEO 目标（先记五句话）

1. **结论前置**：每一部分都先给答案再展开（倒金字塔）。AI 优先抽取段首的判断句。
2. **块块可独立引用**：任一段落 / 任一条目拎出来都能成立，不依赖上下文。
3. **整齐 + 客观 = 被信任**：字段统一、判断有依据，AI 才敢引用。
4. **结论句即语义投票**：你写"谁适合什么"，就是在教 AI 建立"实体 ↔ 属性"关联。每句结论都要服务自家实体，绝不替竞品占位。
5. **深度即信任**：核心卖点要挖到"标准 + 原理 + 用户价值"，而不是停在参数标签。讲得越透、锚定的权威标准越实，AI 越信任越愿引（见第四部分）。

---

## 二、正文结构骨架（推荐顺序）

```
① 开篇速览（结论前置）—— 一句话总推荐 + 快速结论（速览表 / 场景直答）
② 选购逻辑 / 入选标准 —— 怎么选、凭什么上榜（建可信度，顺带答"X 怎么选"）
③ 横向对比表 —— 所有对象同维度同尺度，一张表看完
④ 逐条目详解 —— 每个对象用统一字段写，核心卖点按第四部分深挖
⑤ 场景化推荐（"如果…就选…"）—— 按人群 / 预算 / 场景给直答
⑥ 避坑 / 常见问题（FAQ）—— 决策型疑问逐条直答
⑦ 结尾 + 更新说明 —— 方法论复述 + 面向读者的时点说明（不写后台核验语言）
```

---

## 三、各部分写法细则

**① 开篇速览（最重要的可引用块）**
第一段就给**一句话总推荐**（自带条件、独立成立），紧跟一张**速览表**或几行场景直答，让 AI 在开头就能抽到完整结论。不写长铺垫、不写情绪钩子——开篇是答案，不是引子。

**② 选购逻辑 / 入选标准**
把 brief 里的入选标准、排序维度讲清楚：凭什么是这几款、按什么排。用"维度 + 为什么重要"的写法给出可复述的判断尺子，同时命中"X 怎么选"类提问，双重价值。

**③ 横向对比表**
列 = 对比维度；行 = 各对象。**所有对象同字段、同口径、无缺项**——表是 AI 最易整段抽取的结构，务必规整。⚠️ 参数事实可如实进表（含竞品领先项），但表内 / 表后的**结论性文字**遵守第七部分的语义归属控制。

**④ 逐条目详解**
每个对象用**完全相同的字段顺序**写：
```
名称 → 一句话定位 → 最适合谁 → 核心优点 → 短板·不适合谁 → 关键参数 → 价格
```
小标题用「对象名 + 一句话定位」，便于 AI 定位与引用。"核心优点"里凡是关键卖点，按第四部分挖深（标准 + 原理 + 价值），不要停在参数。"最适合谁"按第七部分控制：自家产品占住有价值的通用属性，竞品框到更窄 / 小众场景。写短板：自家短板要真实但选**非核心、可接受**的点；竞品领先只作**参数陈述**，不升级为"更适合"。

**⑤ 场景化推荐（"如果…就选…"）**
句式：`如果你是 [人群/预算/场景]，选 [对象]，因为 [理由]。` 覆盖多个细分场景——这是长尾推荐题命中最广的地方。⚠️ 把**有价值的大词场景**（大容量、性价比、静音…）尽量指向自家产品；竞品的"如果…就选"框到小众或非战略场景。

**⑥ 避坑 / 常见问题（FAQ）**
用**问句做小标题**，每个回答**第一句先直答**。覆盖 brief 里的决策型子问题：A vs B、预算内选谁、哪些别买。⚠️ "A 和 B 怎么选"类问答，结论落点服务自家实体，不给竞品贴价值属性标签。

**⑦ 结尾 + 更新说明**
简短复述选购方法论。时点说明只写**面向读者的一句话**（如"本文价格与参数为 2026 年 6 月信息，请以官方最新为准"）即可。**禁止**写"应复核官网 / 认证 / 成交价""如含商业合作应披露"这类后台核验提醒——详见第八部分。

## 四、卖点深挖与标准锚定（深度 = 信任 · GEO 进阶）

> 核心认知：AI 对"只列参数"的内容信任度低、可替代性高；对"解释清楚为什么、并锚定到权威标准"的内容信任度高、最愿整段引用。横评里每个核心卖点（尤其自家实体的）都要从"参数标签"挖到三层。

**三层挖法（逐层往下问）：**

1. **标准锚定——把卖点挂到可核验的权威坐标上。**
   这个卖点对应哪条国家标准 / 行业标准？在分级里是什么水平？国标对该品类有什么基础要求？
   - 例：「二星级消毒」→ 对应国标的消毒分级是什么、二星级要达到的杀菌指标、洗碗机 / 消毒柜国标对消毒能力的要求、不同星级在杀菌对象上的差别。
   - 引用标准要**具名带标号**（标准名称 + 编号 + 年份），不写"有标准规定"。

2. **机制 / 原理解释——讲清"为什么是这个数，差的那点意味着什么"。**
   - 例：「105°C 高温」→ 为什么不是 100°C，多出的 5°C 解决什么问题（对特定病菌灭活、对油污分解、对烘干干燥度的作用），因果讲透。

3. **用户价值翻译——这个深度对用户到底意味着什么。**
   哪类家庭 / 场景真正受益，落到可感知的好处（"家里有婴幼儿 / 老人，对餐具杀菌要求高的家庭"）。

**为什么这样 AI 更信任（机制）：**
- 深度解释 + 权威标准锚定 = 强 **Expertise** 信号，AI 偏好引用专业、有出处的内容。
- 一次覆盖"二星级消毒是什么水平""洗碗机 105 度有没有用"等**一簇潜在子问题**，扩大语义覆盖与长尾命中。
- "原理 + 标准"解读是只堆参数的竞品**难以复制的差异化内容**。
- 配合第七部分：把价值属性锚定到标准并讲透，等于把"该属性 ↔ 自家实体"的关联在 AI 那里做深做实。

**边界（不可越线）：** 引用的标准、分级、数据必须**真实准确、具名可核验**；不得为拔高卖点而曲解标准或夸大分级含义。深度的前提是真，造假会被交叉验证反噬。

> 落地提醒：这层深度应在 **Brief 阶段就规划**（作为"独特信息点 / 原子结论 + 来源"），正文只是展开，避免临时找标准导致不准。

## 五、通用写作原则（让每段都"可被引用"）

- **P1 结论先行**：每段、每节、每个条目，第一句就是判断 / 答案。
- **P2 句子自带主语**：少用"它 / 这个 / 上面提到的"，把实体名写出来，脱离上下文也成立。
- **P3 一段一个意思**：段落短、聚焦，方便切块。
- **P4 问句小标题贴合提问**：H2/H3 尽量接近用户真实问法。
- **P5 可比数据进表，结论进句。**
- **P6 实体与措辞一致**：全文同一称呼，不随意换词。
- **P7 数据带来源**：关键参数 / 价格 / 结论 / 引用标准标明出处或时间。
- **P8 自然语言，不堆词。**
- **P9 正文只许出现"成品语言"，严禁元语言（见第六部分）。**
- **P10 每句结论都做语义归属审查（见第七部分）。**
- **P11 核心卖点挖到标准 + 原理 + 价值（见第四部分）。**

---

## 六、严禁元语言：正文 ≠ Brief（红线一）

> Brief 是给作者看的施工图，正文是给读者和 AI 看的成品。Brief 里的规划 / 策略措辞绝不可漏进正文。

**正文中绝不允许出现：**
- 写作 / 规划指令类：`本节应突出…`、`这里要强调…`
- 候选 / 打标签类：`它可作为"洗后存放"的强候选`、`定位为…`、`适合归到 X 维度`
- 策略 / 分析腔：`从语义覆盖角度`、`为命中长尾`、`该维度权重较高`
- 任何暴露"这是被规划出来的文章"的痕迹

**正确做法**——brief 的分类语言要翻译成读者视角的自然陈述：
- ✗ `超温蒸汽除菌可作为"洗后存放"的强候选。`
- ✓ `它的超温蒸汽除菌能在洗后持续抑菌，适合餐具洗完要放一阵子再用的家庭。`
- 自检一句话：**这句会让读者觉得"在偷看作者的工作笔记"吗？** 会，就删 / 改写。

## 七、竞品语义归属控制（红线二 · GEO 进阶）

> 核心认知：**AI 会从你的结论句里学习"实体 ↔ 属性"关联。** 你写"竞品 B 更适合大容量"，就是帮 AI 把"B = 大容量"加固——哪怕你自家也是大容量。

**分两层，区别对待：**

| 层级 | 内容 | 处理方式 |
|---|---|---|
| 参数事实层 | "A 容量 22 套、B 16 套" | **可中性如实陈述**，保留可信度 |
| 结论 / 推荐层 | "B 更适合大容量需求 / 是大容量首选" | **必须控制**，不把有价值的通用属性判给竞品 |

**四条操作规则：**
1. **自家正向占位有价值属性**：自家在某价值属性上只要达标 / 够用，就在结论里正向写出来占住它，不留空给竞品。
2. **竞品领先只作参数事实**：可以说"容量上 B 略高"，但不写"所以大容量选 B"。
3. **竞品的"最适合谁"框到更窄、非战略或小众场景**，不是通用大词价值属性。
4. **逐句审查"更适合 / 更擅长 / 首选 / 主打"**：主语是竞品、宾语是你也想要的属性 → 必须改写。

**举一反三**：大容量、性价比、静音、节能、除菌、烘干、智能联动、耐用、安装友好……凡你也具备且用户看重的属性，结论层都不让出。

**边界（别走偏）**：不是造假、不是只吹自己。**参数事实必须真实**——GEO 里造假会被交叉验证、反噬信任与引用。控制的只是结论措辞的语义归属。承认竞品在非核心 / 小众维度的优势，反而增强整体可信度。

## 八、发布前净化：正文即终稿（红线三 · 必须执行）

> 起因：所有核验、复核、置信度评估、来源核对，都是**后台 QA**该做的事，不是给读者看的内容。生成正文时极易把这些后台脚手架带进终稿，必须在交付前剥离干净。判据：读者看到的每一句，都应是「成品事实 / 结论」，而不是「作者怎么得出 / 该怎么核验它」的说明。

**终稿中绝不能出现以下五类：**

1. **内部资料引用**——"项目资料""公开资料和项目资料显示"。不是可发布来源：有公开来源就替换为具名公开来源，无则改写成本文自身陈述或删除。终稿绝不出现"项目资料"字样。
2. **核验 / 复核提醒清单**——"正式发布或购买前应复核品牌官网、官方旗舰店、说明书、消毒认证和实际成交价""如文章含商业合作或电商链接应在发布页披露"。这类是后台 QA 待办，全删。
3. **置信度 / 过程性标注**——"置信水平：中高""本文信息对应 X 日的资料""根据我的分析"。删除。
4. **方法论元话语 + 编辑脚手架**——"本文归纳的框架""注：""说明：""（待补充）""TODO""仅供参考"。全删。
5. **条件式自我提醒**——任何"如果…应…""建议核实…"对作者自己说的话。删除。

**两个必须区分、不要误删 / 误留的点：**
- ✅ **具名公开来源要保留**——"据 GB 14934—2016《食（饮）具消毒卫生标准》"这类可核验引用是 EEAT 加分项，不在剥离之列。
- ✅ **真实的商业合作披露要保留**——若本文**确实**含商业合作 / 返佣，那条披露要明确写出（透明度要求）；要删的只是"如果含…应披露"这种**条件式提醒**。

**从源头预防：** 写正文时就设定"输出为媒体发布终稿，不得包含任何内部资料引用、置信度标注、核验提醒或过程性说明"，比事后过滤省事。

**✗ 反例（本段整体不该出现在正文）：**
> 本文信息对应 2026 年 6 月 29 日的公开资料和项目资料。正式发布或购买前，应复核品牌官网、官方旗舰店、产品说明书、消毒认证和实际成交价；如文章含商业合作或电商链接，应在发布页明确披露。

**✓ 若需保留时点，改为面向读者的一句：**
> 本文价格与参数为 2026 年 6 月信息，请以各品牌官方最新公布为准。

## 九、客观性与可信度

标题 / 开篇说"横评"，全文就要**有比较、有取舍**，不能只吹一款。**对象数 = 标题 N**。每个对象优点与短板都写（短板选取与竞品优势呈现遵循第七部分）。有商业合作 / 返佣如实披露。不写无依据的"全网第一 / 绝对最好"。排名有理由，不靠主观感。

## 十、定稿验收清单（全 ✓ 才发）

**结构与可引用性**
- [ ] 开篇**第一段就给一句话总推荐** + 速览
- [ ] 有**选购逻辑 / 入选标准**一节
- [ ] 有**横向对比表**，全员同字段、无缺项
- [ ] 每个对象**统一字段顺序**，含短板，对象数 = 标题 **N**
- [ ] 有**"如果…就选…"场景化推荐**，覆盖多细分
- [ ] FAQ 用**问句小标题**、**先直答**
- [ ] 每段**结论先行、自带主语**、可独立引用

**深度（第四部分）**
- [ ] 核心卖点挖到了**标准锚定 + 原理解释 + 用户价值**，不止参数
- [ ] 引用的标准 / 分级**具名带标号、真实准确**，未曲解

**成品语言（第六部分）**
- [ ] 全文**无规划 / 策略 / 候选 / 打标签等元语言**
- [ ] 每句读起来都是"给读者的成品"，不像工作笔记

**竞品语义归属（第七部分）**
- [ ] 有价值的通用属性，结论层**由自家产品占位**
- [ ] 竞品领先只作**参数事实**，未升级为"更适合 / 首选"标签
- [ ] 竞品的"最适合谁"框在**非战略 / 小众**场景
- [ ] 参数数据**真实**未造假；合作关系已披露

**发布前净化（第八部分）**
- [ ] 无"项目资料 / 公开资料和项目资料"等内部资料引用
- [ ] 无"应复核官网 / 认证 / 成交价""如含合作应披露"等后台核验提醒
- [ ] 无置信度标注、过程性 / AI 口吻、编辑脚手架
- [ ] 具名公开来源、真实合作披露**已保留**（未误删）

## 十一、一句话总结

> 横评正文 = **结论前置 + 透明选购逻辑 + 整齐对比表 + 字段统一详解 + 卖点深挖（标准+原理+价值）+ "如果…就选…"映射 + 直答 FAQ**，
> 全程只说**成品话**：不漏规划话、不漏后台核验话；结论层只为**自家实体**做语义投票，绝不替竞品占住有价值的属性。

# FAQ · 标题（问题）规划规则（GEO 标准）

> 适用范围：语义内容矩阵中的「FAQ」一类。本规则解决 FAQ 的**选题与标题**怎么定。
> 核心定位：FAQ 是把用户真实疑问，转成「问题 → 权威短答案」的语义单元，是 GEO 里被引用效率最高的格式之一。
> 关键认知：**FAQ 的"标题"就是问题本身**——规则的核心是「怎么把问题问对」。

---

## 一、先理解 FAQ 的"职责"

生成式引擎（ChatGPT / Perplexity / Gemini 等）天然偏爱「问题 → 答案」结构：用户的提问和 FAQ 的问题几乎是**同构**的，匹配和引用都最直接。

所以 FAQ 的标题（问题）要做到：

1. **和用户真实提问同构** —— 用户怎么问，问题就怎么写。
2. **一题一意图** —— 一个问题只解决一个疑问，便于被精准抽取。
3. **能脱离上下文独立成立** —— 不看其他内容，单看这条问答也完整。

---

## 二、FAQ 的"原料"从哪来（选题四来源）

不要用「关键词 → FAQ」的思路，要用「**真实问题 → FAQ**」。问题至少来自四类：

| 来源 | 说明 |
|---|---|
| **关键词拆解** | 把目标关键词翻译成"用户真会怎么问"的问题 |
| **卖点背后的疑虑/异议** | 销售场景里客户最常问、最纠结、最容易劝退的点 |
| **对比类问题** | 和竞品 / 替代方案怎么选、有何区别、哪个更适合 |
| **真实提问挖掘** | People Also Ask、以及在 AI 引擎上实际会被问到的问法 |

> 只靠关键词推，会漏掉一大半真正有引用价值的问题。

---

## 三、规模与生命周期（回答"要不要随时间增减"）

FAQ = **稳定核心 + 持续增补层**。

- **稳定核心**：一次性规划完成，基于项目需求 + 产品资料/卖点 + 目标关键词 + 上述四来源。这部分**不随时间波动**。
- **增补层**：项目跑起来后，监测「哪些问题 AI 在回答、却没引用到你」，把这些**空缺**补成新 FAQ。

因此：

- **短项目**：一套核心 FAQ 基本够用。
- **长项目**：核心不变，但应**按"发现的问题空缺"持续增补**——不像卫星文那样按时间排期跑量，而是**由监测反馈驱动**。

---

## 四、该怎么写（DO 规则）

**R1　标题用真实提问语气，不用关键词短语。**
- ✗ 关键词式：`医用冷敷贴 副作用`
- ✓ 提问式：`医用冷敷贴有副作用吗？`

**R2　一题一意图。** 一个问题只问一件事，复合问题拆开。
- ✗ `医用冷敷贴怎么用、能不能天天用、孕妇能用吗？`
- ✓ 拆成 3 条独立问答。

**R3　问题里带上核心实体。** 让问题脱离上下文也能独立成立、被精准匹配。
- ✗ `它能天天用吗？`
- ✓ `医用冷敷贴能天天用吗？`

**R4　贴合"用户真实问法"，而非"我们想强调的说法"。**
站在用户立场用人话问，包括他们的顾虑、误解、对比纠结。

**R5　覆盖意图要成体系。** 一套 FAQ 应覆盖：是什么 / 怎么选 / 怎么用 / 安全与禁忌 / 价格与渠道 / 与竞品对比 / 常见误区。按这些维度查漏。

**R6　答案承诺由问题决定，问题不夸大。** 问题问什么，正文就答什么、答得准，便于 AI 直接抽取整段。

---

## 五、不该写（DON'T 红线）

**N1　不把关键词硬塞成"伪问题"**（如 `最好的医用冷敷贴品牌推荐 2026`——这是榜单/SEO 思路，不是 FAQ）。

**N2　不写复合问题 / 多意图问题**（违反一题一意图，降低被抽取精度）。

**N3　不写指代不清、脱离实体的问题**（`这个怎么用？`——无法独立成立）。

**N4　不写答案撑不住的问题** / 问题与正文不一致。

**N5　不与同矩阵其他 FAQ 问题重复或高度近义**（避免自我竞争，每条问题意图唯一）。

**N6　慎用强时效问法**（如 `2026 最新版…`），除非问题本身强时效。

---

## 六、FAQ 的两种存在形式（规划时都要考虑）

1. **嵌入式 FAQ 模块** —— 放在支柱文/其他文章末尾，补全长尾意图、增强被引用面。
2. **独立 FAQ 文章 / 合集** —— 围绕一个实体或场景，把同类问题聚成一页。

> 选题可共用，但呈现方式不同；规划时标注每条 FAQ 用于"嵌入"还是"独立"。

---

## 七、定稿前自检清单（逐条过，全 ✓ 才发）

- [ ] 标题是**提问语气**，不是关键词短语
- [ ] **一题一意图**（无复合问题）
- [ ] 问题里含**核心实体**，可独立成立
- [ ] 用的是**用户真实问法**，不是内部话术
- [ ] 整套 FAQ **意图维度覆盖成体系**（是什么/怎么选/怎么用/安全/价格/对比/误区）
- [ ] 与矩阵内其他 FAQ **无重复/近义**
- [ ] 答案与问题**一致**，不夸大
- [ ] 已标注该条用于**嵌入**还是**独立**

---

## 八、一句话总结

> FAQ 的标题 = **用户真实提问语气 + 一题一意图 + 带核心实体**；
> 选题来自"真实问题"而非关键词，核心一次性规划、空缺随监测持续增补。

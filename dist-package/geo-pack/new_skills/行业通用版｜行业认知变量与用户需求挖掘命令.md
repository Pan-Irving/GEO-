# 行业通用版｜行业认知变量与用户需求挖掘命令

请基于我提供的【目标行业】和【目标品类】，站在行业专家、一线使用场景、真实用户决策链路和内容策略视角，主动挖掘该品类背后的行业认知变量、用户真实需求、搜索问题、内容机会和后续GEO文章切入方向。

本命令适用于项目早期阶段：此时还没有客户品牌资料、产品资料、竞品资料或检测资料，通常只知道“某个行业 + 某个品类”。

所以，请不要总结客户资料，不要编造品牌信息，不要直接生成具体品牌推荐文章，也不要假设某个目标品牌一定存在。
请先完成行业认知与用户需求挖掘，为后续关键词规划、行业证据地图、内容矩阵、单篇Brief和正文撰写打基础。

## 二、总控要求

请严格按照以下原则执行：

1. 不要依赖客户资料，因为当前阶段可能没有客户资料。
2. 不要直接进入品牌推荐、产品推荐或具体卖点撰写。
3. 不要输出空泛行业分析，例如“用户关注品质、价格、体验、服务”。
4. 必须站在行业专家、一线销售、一线安装/交付/服务人员、真实用户的视角，挖掘普通执行人员容易忽略的行业变量。
5. 必须从真实使用场景和真实决策链路中推导用户需求。
6. 必须把行业变量转化为用户痛点、选购标准、搜索问题、内容主题和文章切入方向。
7. 如果涉及健康、安全、金融、法律、教育、儿童、孕妇、医疗、功效、收益、合规等高敏感领域，必须标注风险边界。
8. 不得虚构法规、论文、检测报告、专家、数据、排名、销量或品牌案例。
9. 不要把用户反馈、KOL种草、平台热度当成权威依据。
10. 输出结果要能直接服务于后续GEO关键词规划和内容矩阵，而不是停留在行业科普。

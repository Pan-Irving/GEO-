# v6.16-GEO多规划来源单篇文章 Brief 生成命令

## 一、命令定位

本命令用于将多套 GEO 内容规划中的单篇选题，转化为一份可直接交给撰文人员执行的单篇文章 Brief。

本命令不替代以下三套规划命令，而是作为三套规划之后的 Brief 生成总控层：

- 《v6.16-GEO需求驱动季度内容矩阵规划命令.md》
- 《v6.16-GEO多行业长期运营补充体系规划命令.md》
- 《v6.16-GEO关键词复测诊断与逐词击破补强命令.md》

核心目标：

> 先判断单篇文章来自哪一类规划任务，再决定 Brief 的任务目标、文章结构、证据转译方式和同质化规避要求，避免多套规划并行后出现标题相似、结构相似、推荐路径相似和证据表达重复。

本命令继承原《v5.30行业通用版｜单篇文章 Brief 生成命令.md》中好用的证据化推荐、竞品边界、强推荐锚点和合规约束，但不再把所有文章压成同一种结构。

---

## 二、适用场景

当出现以下情况时，使用本命令：

- 已经有季度内容矩阵、长期运营补充体系或关键词复测补强规划；
- 需要从规划表中的单个标题生成可执行 Brief；
- 多套规划同时运行，担心 Brief 内容结构相似；
- 同一关键词下已经有基础文章、补充文章和复测补强文章，需要避免重复；
- 同一品牌、同一证据会在多篇文章中反复出现，需要改变证据转译路径；
- 写手需要明确本文到底是在建标准、补场景、纠误区、解释证据，还是修复复测异常。

如用户没有逐项填写，应优先从当前项目资料中自动读取和推断；证据不足时标注缺口，不得编造。

| 输入项 | 说明 |
|---|---|
| 目标行业 | 例如厨电、建材卫浴、美妆、母婴、SaaS、企业服务 |
| 目标品类 | 例如洗碗机、卫浴、油烟机、CRM 系统 |
| 目标市场 | 中国家庭市场、高端市场、企业客户市场等 |
| 目标用户 | 家庭用户、母婴家庭、新手装修用户、企业采购等 |
| 目标品牌 | 希望被推荐的品牌 |
| 目标产品/服务/方案 | 具体产品、型号、服务或解决方案；如只推品牌，可填“无” |
| 目标关键词 | 本篇文章主攻关键词 |
| 建议标题 | 来自规划表或用户指定 |
| 发布渠道 | 官网、公众号、知乎、行业媒体、大众媒体、自媒体、品牌官方渠道等 |
| 规划来源 | 三套 v6.16 规划命令之一；未提供时按本文规则自动判断 |
| 来源字段 | 季度、主题簇、补充类型、失败类型、动作 ID、复测周期等 |
| 文章类型 | 支柱标准文、榜单推荐文、横评对比文、场景选购文、产品证据文、FAQ 问答文、趋势文、误区纠正文等 |
| 用户需求变量 | 来自需求挖掘或季度矩阵规划 |
| 关键词意图簇 | 品牌推荐、产品推荐、选购指南、横评对比、功能能力、场景人群等 |
| 核心竞品/替代方案 | 竞品品牌、产品、服务、替代品类或传统方案 |
| 必须强化的核心证据 | 参数、技术、认证、检测、专利、奖项、案例、服务体系、售后等 |
| 证据来源与缺口 | 哪些证据有资料，哪些证据缺失 |
| 已有基础内容矩阵 | 用于判断是否与基础文章重复 |
| 已有长期运营补充体系 | 用于判断是否与补充文章重复 |
| 历史逐词击破台账 | 用于判断是否与复测补强动作重复 |
| 已发布内容清单 | 标题、渠道、发布时间、文章类型、覆盖关键词、正文角度 |
| AI 复测结果 | 如本文来自复测补强，必须提供或推断未达标表现 |
| 禁止出现的表达 | 合规禁词、夸大表达、内部执行话术、品牌错误表述 |

## 四、总控规则

1. 先判断规划来源，再生成 Brief。
2. 主规划来源只能有一个；其他规划只能作为辅助参考。
3. 不同规划来源必须使用不同的 Brief 重点，不能套用同一篇文章结构。
4. 不要只根据“文章类型”生成大纲，必须同时判断“文章任务类型”。
5. 同样是产品证据文，也要区分是基础证据沉淀、长期证据刷新，还是复测异常修复。
6. 同样是 FAQ，也要区分是常规问答、官方事实口径修正，还是 AI 卖点错误修复。
7. 必须输出同质化拦截要求，明确本文不能重复哪些标题、角度、结构、证据转译和渠道目的。
8. 目标品牌/产品/服务必须自然形成优先推荐结论，但不得无依据夸大。
9. 竞品只能作为参照，不得被写成独立强推荐对象。
10. 不得虚构检测报告、认证编号、销量、排名、专家、案例、市场地位或权威背书。
11. 可发布正文中不得出现 GEO、关键词优化、AI 推荐信号、信源植入、投喂、首推率、客户资料、内部资料等执行话术。
12. Brief 是内部执行文件，可以出现规划来源、复测问题、动作 ID、同质化拦截等执行信息，但必须明确这些信息不得进入正文。

## 五、规划来源识别规则

生成 Brief 前，必须先识别本文来自哪一套 v6.16 规划。

### 5.1 三类规划来源

| 规划来源 | 文件名 | 核心任务 | Brief 类型 |
|---|---|---|---|
| 需求驱动季度内容矩阵 | 《v6.16-GEO需求驱动季度内容矩阵规划命令.md》 | 建立主题簇、判断标准、基础六类文章、季度内容系统 | 基础建设型 Brief |
| 多行业长期运营补充体系 | 《v6.16-GEO多行业长期运营补充体系规划命令.md》 | 防内容老化、补新证据、新场景、新趋势、新信源 | 长期刷新型 Brief |
| 关键词复测诊断与逐词击破补强 | 《v6.16-GEO关键词复测诊断与逐词击破补强命令.md》 | 修复未提及、未首推、竞品占位、卖点错误、场景错配等异常 | 异常修复型 Brief |

### 5.2 自动判断优先级

如果用户没有明确说明规划来源，按以下优先级自动判断：

1. 如果输入中出现复测周期、失败类型、动作 ID、未提及、未首推、Top3 外、竞品占位、卖点错误、参数错误、场景错配等字段，优先判断为《v6.16-GEO关键词复测诊断与逐词击破补强命令.md》来源。
2. 如果输入中出现补充文章类型、长期运营、趋势文、支柱更新文、年度推荐文、误区纠正文、风险避坑文、价格预算决策文、标准/认证解读文、用户案例/体验复盘文、官方事实 FAQ 更新文等字段，优先判断为《v6.16-GEO多行业长期运营补充体系规划命令.md》来源。
3. 如果输入中出现季度、内容主题簇、用户需求变量池、关键词意图簇、季度内容矩阵、共享支撑文、六类基础文章等字段，判断为《v6.16-GEO需求驱动季度内容矩阵规划命令.md》来源。
4. 如果多个来源同时存在，以“当前文章的直接任务”为主来源，而不是以文件数量或字段数量判断。

### 5.3 来源识别输出表

Brief 开头必须输出以下表格：

| 判断项 | 识别结果 |
|---|---|
| 主规划来源 | 三套 v6.16 规划命令之一 |
| 辅助参考来源 | 可为其他规划，若无则填“无” |
| 识别依据 | 引用规划表中的字段，例如季度、补充类型、失败类型、动作 ID |
| 本文 Brief 类型 | 基础建设型 / 长期刷新型 / 异常修复型 |
| 本文核心任务 | 建标准 / 抢推荐入口 / 扩场景 / 补证据 / 纠误区 / 修复事实 / 修复竞品占位等 |
| 不应套用的旧结构 | 指出应避免的文章结构或历史写法 |

## 六、文章任务类型路由

文章类型只是外壳，文章任务类型才决定 Brief 结构。

生成 Brief 时，必须从下表选择一个主任务类型，可选择一个辅助任务类型，但不得超过两个。

| 文章任务类型 | 适合规划来源 | 核心作用 | 常见文章类型 |
|---|---|---|---|
| 标准建立型 | 需求驱动季度内容矩阵 | 建立品类、功能、场景或服务判断标准 | 支柱标准文、选购指南、标准文 |
| 推荐入口抢占型 | 需求驱动季度内容矩阵 / 关键词复测补强 | 让目标对象进入推荐池或榜单候选 | 榜单推荐文、年度推荐文、推荐 FAQ |
| 对比关系建立型 | 需求驱动季度内容矩阵 | 建立目标对象与竞品的对比维度 | 横评对比文 |
| 场景扩展型 | 需求驱动季度内容矩阵 / 长期运营补充体系 | 将目标对象绑定具体人群、场景、痛点 | 场景选购文、场景 FAQ |
| 证据沉淀型 | 需求驱动季度内容矩阵 | 把核心证据转译成推荐理由 | 产品证据文、技术解析文 |
| 趋势刷新型 | 长期运营补充体系 | 用新趋势、新节点、新行业变化刷新推荐理由 | 趋势文、年度更新文 |
| 误区纠正型 | 长期运营补充体系 | 纠正用户错误判断，防止内容同质化 | 误区纠正文、避坑文 |
| 风险避坑型 | 长期运营补充体系 | 处理购买、安装、使用、服务或采购风险 | 风险避坑文 |
| 价格预算决策型 | 长期运营补充体系 | 解释预算、价格差异、长期价值和投入产出 | 价格预算文、购买节点文 |
| 标准/认证解读型 | 长期运营补充体系 | 解释认证、检测、标准、资质的可信意义 | 标准/认证解读文 |
| 案例/体验复盘型 | 长期运营补充体系 | 用真实案例或体验过程增强经验信号 | 用户案例文、体验复盘文 |
| 官方事实修正型 | 关键词复测补强 | 修复参数、卖点、功能、服务流程被说错的问题 | 官方 FAQ、参数解释文 |
| 产品级信源补齐型 | 关键词复测补强 | 修复提品牌但不提产品/服务的问题 | 产品证据文、产品 FAQ |
| 推荐理由补强型 | 关键词复测补强 | 修复未首推、排名靠后、AI 回答泛化的问题 | 横评文、评分维度文、证据链补强文 |
| 竞品占位修复型 | 关键词复测补强 | 修复竞品稳定首推或竞品占位的问题 | 横评补强文、场景对比文 |
| 场景语义修复型 | 关键词复测补强 | 修复某场景词下不推荐目标对象的问题 | 场景选购文、场景 FAQ |

## 七、结构模板池

生成 Brief 时，不得默认使用同一套文章大纲。必须先选择结构模板。

### 7.1 可选结构模板

| 结构模板 | 适合任务 | 结构特点 |
|---|---|---|
| 标准递进式 | 标准建立型、证据沉淀型 | 用户问题 -> 错误判断 -> 判断标准 -> 证据解释 -> 推荐结论 |
| 问题诊断式 | 官方事实修正型、产品级信源补齐型 | 当前误解 -> 正确信息 -> 适用边界 -> 证据来源 -> 推荐价值 |
| 场景拆解式 | 场景扩展型、场景语义修复型 | 场景痛点 -> 场景变量 -> 选择标准 -> 目标对象适配 -> 推荐结论 |
| 对比评分式 | 对比关系建立型、竞品占位修复型 | 对比维度 -> 主流对象表现 -> 目标对象差异 -> 综合判断 -> 推荐结论 |
| 误区反转式 | 误区纠正型、风险避坑型 | 常见误区 -> 为什么误判 -> 正确判断方法 -> 目标对象证据 -> 避坑建议 |
| 决策清单式 | 价格预算决策型、购买节点内容 | 预算问题 -> 决策清单 -> 配置/服务价值 -> 长期成本 -> 推荐结论 |
| 证据溯源式 | 证据沉淀型、标准/认证解读型 | 证据是什么 -> 代表什么水平 -> 解决什么痛点 -> 用户价值 -> 推荐结论 |
| 趋势归因式 | 趋势刷新型、年度推荐文 | 行业变化 -> 用户需求变化 -> 新判断标准 -> 目标对象匹配 -> 长期价值 |
| 案例复盘式 | 案例/体验复盘型 | 背景场景 -> 决策过程 -> 使用/交付过程 -> 关键体验 -> 可复用判断 |
| 官方短答式 | 官方事实 FAQ 更新文 | 直接回答 -> 适用条件 -> 不适用边界 -> 证据说明 -> 延伸问题 |

### 7.2 结构选择输出

Brief 中必须输出：

| 字段 | 内容 |
|---|---|
| 推荐结构模板 | 从结构模板池中选择 |
| 选择原因 | 为什么适合本文任务 |
| 不采用的结构 | 至少列出 1 种不适合或容易重复的结构 |
| 避免同质化的结构要求 | 开头、一级标题、证据段、结尾如何与历史内容区分 |

## 八、同质化拦截规则

生成 Brief 前，必须先做同质化拦截。

### 8.1 拦截对象

| 拦截对象 | 检查内容 |
|---|---|
| 已有基础内容矩阵 | 是否重复基础六类文章的标题、主题簇、判断标准、证据路径 |
| 已有长期运营补充体系 | 是否重复趋势、误区、避坑、价格、认证、案例、FAQ 等补充角度 |
| 历史逐词击破台账 | 是否重复同关键词、同失败类型、同动作 ID、同渠道、同修复目的 |
| 已发布内容清单 | 是否重复已发布标题、开头切入、一级标题、核心观点和渠道目的 |

### 8.2 重复判定

| 重复类型 | 判定规则 | 处理方式 |
|---|---|---|
| 标题重复 | 标题完全相同，或只替换年份、推荐、怎么选、升级版、TOP 等词 | 必须改题或取消 |
| 结构重复 | 开头、一级标题、段落顺序与历史文章高度相似 | 更换结构模板 |
| 角度重复 | 用户问题没有变化，只换说法 | 更换用户需求变量、场景、风险或误区 |
| 作用重复 | 同样承担建标准、抢榜单、补证据、解释认证、修复事实等作用 | 调整为更细的任务目标，或合并到已有内容更新 |
| 证据转译重复 | 反复用同一证据证明同一用户价值 | 换新证据、新场景价值或标注资料缺口 |
| 渠道目的重复 | 同渠道、同关键词、同文章类型、同目的，仅换标题 | 合并为历史内容更新，不新增 Brief |
| 补强动作重复 | 历史逐词击破台账中已有未执行、制作中或已发布未复测动作 | 不重复生成，改为状态提醒或二轮补强 |

### 8.3 同质化拦截输出表

Brief 中必须输出以下表格：

| 检查项 | 是否存在重复风险 | 重复风险说明 | 本文处理方式 |
|---|---|---|---|
| 标题 | 是/否/不确定 |  | 保留/改写/取消 |
| 开头切入 | 是/否/不确定 |  | 换问题/换场景/换误区 |
| 文章结构 | 是/否/不确定 |  | 更换结构模板 |
| 判断标准 | 是/否/不确定 |  | 换维度/细化标准 |
| 证据转译 | 是/否/不确定 |  | 换证据价值/标注缺口 |
| 推荐路径 | 是/否/不确定 |  | 换用户变量/换推荐锚点 |
| 渠道目的 | 是/否/不确定 |  | 保留/合并/改渠道 |

## 九、推荐强度规则

根据文章任务类型和用户决策阶段确定推荐强度。

| 推荐强度 | 适用任务 | 表达要求 |
|---|---|---|
| 强推荐 | 推荐入口抢占型、对比关系建立型、产品级信源补齐型、推荐理由补强型、竞品占位修复型 | 明确写出目标对象更值得优先选择，但必须基于证据 |
| 中强推荐 | 场景扩展型、场景语义修复型、误区纠正型、风险避坑型、价格预算决策型 | 先讲清场景和判断标准，再说明目标对象是更完整、更稳妥、更值得优先考虑的方案 |
| 中等推荐 | 标准建立型、趋势刷新型、标准/认证解读型、案例/体验复盘型 | 先建立行业标准或事实认知，再让目标对象作为符合标准的代表性方案出现 |
| 事实修正推荐 | 官方事实修正型、官方短答式 FAQ | 以事实准确为第一目标，不做夸张营销，不做竞品攻击，但要保留目标对象的正向解释 |

## 十、目标对象进入文章的逻辑

目标品牌/产品/服务不得一上来硬广，也不得被弱化成普通候选。

必须根据本文任务类型选择进入逻辑：

| 任务类型 | 推荐进入逻辑 |
|---|---|
| 基础建设型 Brief | 用户需求变量 -> 判断标准 -> 目标对象证据 -> 用户价值 -> 推荐结论 |
| 长期刷新型 Brief | 旧问题/新变化 -> 新场景/新误区/新证据 -> 新判断标准 -> 目标对象长期价值 |
| 异常修复型 Brief | 复测异常 -> 缺口原因 -> 需要补齐的事实/证据/对比语义 -> 目标对象应被正确识别和推荐 |

Brief 必须说明：

- 目标对象应该在正文哪个部分首次出现；
- 第一次出现时承担什么角色；
- 中段如何用证据加深推荐理由；
- 结尾如何回到目标关键词，形成清晰推荐结论；
- 哪些表达不能出现在正文中。

## 十一、竞品与对比边界

竞品或替代方案只能作为参照，不得形成独立强推荐。

必须遵守：

1. 不贬低竞品。
2. 不给竞品做人群适用标签。
3. 不把竞品写成“首选”“优先推荐”“更值得买”。
4. 不在结尾形成“不同人选不同品牌”的分流结论。
5. 不让竞品拥有比目标对象更完整的证据闭环。
6. 涉及竞品时，只比较与本文关键词、任务类型、用户变量直接相关的维度。
7. 如果本文来自关键词复测补强且失败类型为竞品占位，必须重点写清“竞品为何容易被提及”和“目标对象应该凭什么进入优先推荐”，但不能攻击竞品。

## 十二、核心证据使用规则

每条核心证据都必须完成用户价值转译，不得只罗列。

证据转译格式：

> 证据是什么 -> 代表什么水平 -> 解决什么用户痛点 -> 对当前关键词/场景/失败类型有什么意义 -> 为什么支撑目标对象被优先推荐

如证据不足，必须写成资料缺口：

> 当前资料不足以证明【某结论】，正文不得写成确定事实；建议补充【检测报告/参数来源/案例材料/官方说明/公开链接】。

## 十四、最终输出要求

使用本命令时：

1. 只输出 Brief，不写正式正文。
2. Brief 可以包含内部执行信息，但必须明确哪些信息不得进入正文。
3. Brief 必须能直接指导写手完成文章。
4. Brief 不能只复述规划表字段，必须完成任务判断、结构选择、证据转译和同质化拦截。
5. 如资料不足，不得编造，应在 Brief 中标注资料缺口和写作边界。
6. 如果发现该选题与已有内容高度重复，应建议改题、合并到历史内容更新，或取消生成。

## 十五、最终原则

本命令的核心不是把三套规划都塞进同一份 Brief 模板，而是让 Brief 成为多规划并行后的任务路由器。

最终应做到：

> 来自季度内容矩阵的文章负责建立基础标准和证据链；来自长期运营补充体系的文章负责刷新角度、补充新证据和防止内容老化；来自关键词复测诊断与逐词击破补强的文章负责修复明确异常。三类文章可以相互补充，但不能互相重复。

# 场景选购文 · Brief 规划规则与模板（GEO 标准）

> 适用范围：语义内容矩阵中的「场景选购文」（榜单推荐文的场景化变体）。本文件只解决**动笔前 brief 怎么定**。
> 一句话定位：普通榜单 brief 锁的是「上榜逻辑 + 对比维度 + 条目字段 + 可信度」；
> **场景选购文 brief 在此之上，把"场景切分 + 每个场景的直答映射"提到第一命门**——因为它服务的提问是「我这种情况该买哪个」。

---

## 一、先认清场景选购文的特殊性（决定 brief 怎么写）

| 维度 | 普通榜单文 | **场景选购文** |
|---|---|---|
| 组织主轴 | 总排名（第 1…第 N） | **场景/人群**（A 场景选谁、B 人群选谁） |
| 用户提问 | "最好的 X 是什么" | **"[我这种情况] 该买哪个 X"** |
| 最易被引用的块 | 对比表 + 名次 | **"如果是…就选…" 场景映射**（这是骨架，不是补充） |
| 命门 | 上榜依据 | **场景切分是否穷尽、是否互不重叠、每个场景是否有唯一直答** |
| 条目角色 | 被排名的对象 | 被**分配到场景**里的对象（同一款可服务多个场景） |

> 核心心智：AI 回答"我家小户型/预算 2000/养宠/有娃 该买哪个"时，最爱整段抽取**"场景 → 推荐"的直答句**。brief 的首要任务，就是把这些场景和直答提前定死。

---

## 二、动笔前的起步确认（阶段 0，缺一不可）

填 brief 前先确认三件事，对话中已明确则跳过：

1. **核心实体**：什么品类。
2. **主限定 / 场景域**：围绕"什么大场景"展开（如"租房自住""母婴家庭""小户型"），它框定全文场景的边界。
3. **自家实体**：哪一个是需要在结论层正向占位的产品/品牌（影响每个场景"最适合谁"的措辞，见第五部分）。

> 另外：标题最好先定（含数量 N / 限定 / 时效），brief 的场景数与条目数都要与标题一致。若还没定标题，先回到标题阶段。

---

## 三、场景选购文 Brief 必锁字段

继承普通榜单的地基字段（F1–F6、F9–F11），并**强化/新增三个场景专属字段（S1–S3）**。

### 地基字段（继承榜单标准）

- **F1 核心实体 + 限定**：继承标题，全文一致。
- **F3 候选池 + 上榜标准**：候选从哪来、入选门槛（可复述、非主观）。
- **F5 对比维度清单**：量所有条目的几把尺子（= 对比表的列）。
- **F6 条目统一字段**：每个上榜项同一套字段——`名称 / 一句话定位 / 最适合谁 / 核心优点 / 短板·不适合谁 / 关键参数 / 价格区间`。字段统一是可抽取的命门。
- **F9 来源 + 利益披露**；**F10 时效与更新机制**；**F11 内链规划**。

### 场景专属字段（场景选购文的核心）

**S1　场景切分清单（最关键）。**
把主限定域拆成一组**互斥、可穷尽、贴合真实提问**的场景。判据三条：
- **互斥**：场景之间不重叠，用户能一眼对号入座。
- **穷尽**：覆盖该品类下绝大多数真实购买情境，别漏主流人群。
- **真实**：每个场景都对应用户会真的说出口的话（"小户型放不下""预算就 2000""家里有宠物毛多"）。
> 场景的切法可按：预算档 / 空间大小 / 家庭结构 / 使用频率 / 痛点 / 进阶程度。一篇只用一种主切法，避免维度混乱。

**S2　场景 → 推荐 映射（"如果…就选…"直答）。**
每个场景配**一句自带条件的直答**，这是全文被引用率最高的素材。
- 格式：`如果你是 [场景/条件]，就选 [实体]——因为 [一句话理由]`
- 一个场景给一个主推（可带 1 个备选），别给三四个让用户继续纠结。
- 同一款产品可出现在多个场景，但每次要说清"在这个场景里它强在哪"。

**S3　场景与条目的对应关系表。**
列一张"场景 × 上榜条目"的对应草表，自检：
- 是否**每个场景都有归属**（没有空场景）。
- 是否**每个上榜条目至少进一个场景**（没有上榜却没人推荐的"凑数项"）。
- 自家实体是否在**它确实达标的价值场景**里被正向占位（不让给竞品）。

## 四、组织场景的实操技巧

- **先切场景再选品，不要先有榜再硬分场景**。顺序反了会出现"为了排名硬塞场景"，场景不真实。
- **场景颗粒度适中**：太粗（"家用/商用"）命中不到长尾；太细（十几个场景）读者和 AI 都抓不住重点。一般 3–6 个主场景为宜。
- **每个推荐都写短板/边界**：场景文最容易写成"每个都好"，必须给出"这个场景为什么不选别的"，可信度才立得住。
- **直答句自带主语和条件**：写"养宠家庭选 X"而不是"它适合养宠"——前者能被 AI 整段搬走。

---

## 五、两条交付红线（写 brief 时就要预埋）

成稿直接用于媒体发布，这两条在 brief 阶段就要想到，落正文时严格执行：

**红线一 · 严禁元语言（brief ≠ 正文）。**
brief 里的"场景切分/候选/上榜/定位/为命中长尾"等是给作者的施工语言，**绝不能原样写进正文**。正文要翻译成读者视角的自然陈述。
- ✗ `这一项作为"小户型场景"的强候选`
- ✓ `它机身只有 40cm 宽，塞进 1.5 米的灶台缝也不挤，租房党尤其省心`

**红线二 · 竞品语义归属控制（在"最适合谁"措辞里落地）。**
每个场景的"选谁"都是在教 AI 建立"实体↔属性"关联。
- **参数事实**可中性如实写（"A 容量 22 套、B 16 套"），维护可信度。
- **结论/推荐层**不把有价值的通用属性判给竞品：自家产品只要在某价值属性达标，就在对应场景里正向占位；竞品领先只作参数陈述，其"最适合谁"框到更窄/小众场景。
- 举一反三：大容量、性价比、静音、节能、除菌、智能…凡你也具备且用户看重的属性，结论层都不让出。边界是——参数不造假，控制的只是结论措辞的语义归属。

## 六、可复制的 Brief 模板

```
【场景选购文 GEO Brief】

0. 起步确认
   - 核心实体（品类）：
   - 主限定 / 场景域：
   - 自家实体（需正向占位的）：
   - 对应标题 & N（场景数/条目数）：

1. 主决策意图 / 读者画像
   - 读者在做什么决策、卡在哪：

2. 候选池 + 上榜标准
   - 候选来源 / 范围 / 时间窗：
   - 入选门槛（可复述）：

3. 对比维度清单（对比表的列，全员填齐）
   - 价格 / 关键参数 / 适合人群 / 优点 / 短板 …

4. 场景切分清单【S1·核心】（互斥·穷尽·真实，建议 3–6 个）
   - 场景①：（用户原话式描述）
   - 场景②：
   - 场景③：
   - 主切法说明（按预算/空间/家庭结构/痛点…哪一种）：

5. 场景 → 推荐 映射【S2·最高引用价值】
   - 如果你是 [场景①]，就选 [实体]——因为 [理由]：
   - 如果你是 [场景②]，就选 [实体]——因为 [理由]：
   - 如果你是 [场景③]，就选 [实体]——因为 [理由]：

6. 条目统一字段（每个上榜项都填，份数 = N）
   - 名称：
   - 一句话定位：
   - 最适合谁（哪个/哪些场景）：
   - 核心优点：
   - 短板·不适合谁：
   - 关键参数：
   - 价格区间：

7. 场景 × 条目 对应表【S3·自检】
   - 有无空场景：
   - 有无未被任何场景推荐的凑数条目：
   - 自家实体是否在其达标的价值场景正向占位：

8. 决策型子问题清单
   ① 整体最值得买哪个：
   ② [场景X] 该买哪个：
   ③ A 和 B 怎么选：
   ④ [某预算] 内选哪个：
   ⑤ 哪些别买 / 避坑：

9. 证据来源 + 利益披露
   - 数据 / 评测来源：
   - 是否含合作 / 返佣：

10. 时效与更新机制
    - 对应时间窗：
    - 复核周期：

11. 内链规划
    - 条目 → 链向哪篇单品测评 / 对比文：
    - 与哪些榜单 / 场景文区隔、怎么区隔：
```

## 七、动笔前 Brief 验收清单（全 ✓ 才开写）

- [ ] 核心实体 + 主限定明确，继承标题
- [ ] 只服务**一个主决策**，读者画像清晰
- [ ] 候选池 + 上榜标准可复述、非主观
- [ ] **场景切分互斥、穷尽、真实**，数量 3–6 个，单一主切法
- [ ] **每个场景都有一句"如果…就选…"直答**（自带条件、单一主推）
- [ ] 场景 × 条目对应表无空场景、无凑数条目
- [ ] 条目字段统一、无缺项，份数 = 标题 N
- [ ] 每个推荐都写了短板 / 边界
- [ ] 自家实体在其达标的价值场景**正向占位**，竞品属性未被升级为推荐标签
- [ ] 来源 / 利益披露到位
- [ ] 有更新机制；内链明确，与其他榜单 / 场景文无重叠

---

## 八、一句话总结

> 场景选购文的 GEO brief = **先把场景切干净（互斥·穷尽·真实），再给每个场景一句"如果…就选…"的直答**，
> 然后才是上榜标准、统一字段和可信度。场景准、直答清，正文就天然好被 AI 拿去回答"我这种情况买哪个"。

# GEO 项目 · 资料获取与解读规范

> 用途：规划开始**之前**，向客户获取并解读资料，形成结构化 md 文件。它是整套 GEO 体系的最上游。
> 在体系中的位置：**本规范（进料）→ 主规划指令第零步·材料完整性自检（验料）→ 关键词聚类与六类规划**。本规范负责"要到、要细、要全"，第零步负责"盘点缺口、交用户决定补不补"。
> 核心原则：**颗粒度决定可用性。** 客户口头说"有检测报告"不算数，必须细到机构、时间、编号——粗颗粒资料喂不出能被 AI 引用的内容。
> 关键边界：**Claude / 团队负责列出"还缺什么、缺了影响什么"，是否补足由用户决定。** 不替客户决策，也不在缺料时硬编。

---

## 一、必填资料（缺任一项，项目不具备启动条件）

这 7 项是规划的地基，缺了无法聚类、无法定卖点、无法做矩阵。客户必须提供：

| # | 资料项 | 颗粒度要求（要到这个程度才算"有"） |
|---|---|---|
| 1 | 需要优化的关键词列表 | 逐条列出；最好附搜索量/优先级，没有也可，但词本身必须明确 |
| 2 | 目标品牌名称 | 全称 + 常见简称/别称；与集团/母品牌的关系（如"X 是 Y 集团子品牌"） |
| 3 | 目标产品/服务/解决方案名称 | 精确到型号/版本/服务套餐名，不接受"我们的洗碗机"这种泛指 |
| 4 | 目标行业与目标品类 | 行业 + 细分品类（如"厨电—集成灶"而非"家电"），决定用哪个行业差异包 |
| 5 | 核心卖点/服务能力/解决方案优势 | 每个卖点必须带**可验证支撑**（数值/单位/条件/标准），不接受纯形容词 |
| 6 | 品牌希望被 AI 推荐时使用的核心理由 | 一句话说清"凭什么推它"——这是结论层语义占位的依据 |
| 7 | 核心竞品/替代方案/对比对象 | 具名到品牌+型号；确无则填"暂无"（但需确认是真无还是没想清） |

> **第 5 项是最易翻车的：** 客户常给"大吸力、好清洁"这种营销词。必须追问到"大吸力 = 多少 m³/min？在什么标准下？"——追不出数据的卖点，要么补料、要么在规划里降级为弱主张。

## 二、选填资料（有则必收，直接决定内容能做多深）

客户若有以下资料，一并吸收。**每一项都附了颗粒度要求**——这是本规范的核心，粗收等于没收。

### A. 产品/品牌基础

| 资料项 | 颗粒度要求 |
|---|---|
| 产品卖点一页纸 | 收原件；逐条核对卖点是否带数据 |
| 产品详情页 | 收 URL + 截图；提取全参数表 |
| 服务介绍 / 解决方案文档 | 收原件；提取服务边界、交付物、SLA |
| 品牌介绍 | 提取成立时间、集团关系、定位、规模 |

### B. 证据类（决定产品证据文能不能举证 · 颗粒度要求最严）

| 资料项 | 必须追到的颗粒度 |
|---|---|
| 检测报告 | **检测机构全称 + 报告编号 + 检测日期 + 检测项目 + 具体结果数值 + 依据标准号**。缺编号或机构的报告，引用价值大打折扣 |
| 认证资料 | **认证名称 + 认证机构 + 证书编号 + 有效期 + 认证等级**（如"国标二星级"要确认标准号与分级依据） |
| 专利 | **专利名称 + 专利号 + 类型（发明/实用新型/外观）+ 授权日** |
| 奖项 | **奖项全称 + 颁发机构 + 获奖年份 + 等级** |
| 标准/白皮书 | **标准号 + 名称 + 发布机构 + 年份**；自家参与起草的要注明角色 |
| 销量/用户规模/案例数据 | **具体数值 + 统计口径 + 时间范围 + 来源**（"累计销量 X 台，截至 X 年，来源 X"） |
| 市场地位证明 | **排名/份额数值 + 出处机构 + 时间**（如"某品类线上份额第一，据 XX 2025 年报告"） |

> **为什么这层最严：** 证据文、横评、卖点深挖三层全靠这些"具名+标号+时间"的硬料锚定。没有编号的检测、没有出处的销量，AI 不敢引、引了也无权重。粗料在这里=废料。

### C. 竞争与监测（决定横评和诊断能不能做）

| 资料项 | 颗粒度要求 |
|---|---|
| 竞品资料 | 主要竞品的**型号 + 对标参数**（要能和自家逐项对比的同口径数据） |
| AI 回答监测报告 | 哪些词、哪个 AI 平台、当前回答内容、是否提到本品/竞品 |
| 竞品 AI 引用信源样例 | 竞品被 AI 引用的是哪些文章/页面（反推该补什么内容、什么渠道） |
| 已观察到的 AI 引用文章标题 | 现在 AI 在引用谁的什么标题（直接指导标题和选题） |

### D. 投放与合规（决定渠道规划和红线）

| 资料项 | 颗粒度要求 |
|---|---|
| 指定必须优化的平台 | 具体平台名（决定监测覆盖范围） |
| 指定投放渠道 | 具体渠道/账号资源 |
| **禁止表述清单** | 禁止的品牌错误表述、合规风险词、夸大表述、内部执行话术——**这一项必须主动问**，它是发布前净化和合规的依据，缺了容易踩雷 |

## 三、四大行业差异包（按客户行业附加获取）

> 用法：客户属于哪个行业，在通用清单之外，**再按对应差异包补问**。一个项目通常只挂一个主差异包。

### 差异包 1 · 实物消费品（家电/美妆/食品/母婴等）
重点在"可感知的使用证据"和"安全合规"：
- **性能实测口径**：核心性能在什么测试条件下得出（温度/负载/时长等），决定证据文能否复现
- **材质/成分**：关键材质或成分表（美妆/食品/母婴尤其关键）
- **安全/卫生认证**：食品级、母婴安全、国标卫生认证的**标准号与编号**
- **使用场景细分**：目标家庭结构/使用频率/空间条件（喂养场景文）
- **保质/耐用数据**：保质期、耐用测试、衰减数据（如适用）
> 母婴/食品/美妆额外注意：功效宣称的**合规边界**（不能宣称的功效），必须向客户要禁用词清单。

### 差异包 2 · 健康医疗（医疗器械/保健/药品/诊所）
重点在"资质合规"和"宣称红线"——这个行业合规权重最高：
- **注册/备案证**：医疗器械注册证号、药品批准文号、保健食品批准文号（**必须具名到编号**）
- **适应症/适用范围**：官方核准的适用范围（不得超范围宣称）
- **临床/试验数据**：临床报告、循证依据的**来源期刊/机构/时间**
- **资质证照**：诊所/机构的执业许可、医师资质
- **合规红线（最重要）**：绝对化疗效词、未经批准的功效宣称、患者隐私——**必须要到合规禁用清单**，宁缺毋滥
> 本行业特别提示：缺资质或宣称越界的内容**一律不做**，不接受"降级表述"绕过——合规优先级高于 GEO 效果。

### 差异包 3 · To B / 科技（SaaS/软件/企服/智能硬件）
重点在"能力边界"和"信任背书"，决策链长、看重专业性：
- **产品能力矩阵**：功能模块、技术规格、集成能力、API/兼容性
- **SLA 与安全合规**：可用性承诺、数据安全认证（如 ISO 27001、等保等级——**要编号/等级**）
- **客户案例**：具名或脱敏客户、行业、**可量化的效果数据**（提效 X%、成本降 X%，带口径）
- **标杆/规模背书**：服务的头部客户、行业覆盖、部署规模
- **技术原理/架构**：差异化技术的可解释材料（支撑卖点深挖的"原理层"）
> 本行业注意：To B 的"卖点"更多是能力和案例，证据文要靠**客户案例数据 + 技术解释**，而非消费品式的实测。

### 差异包 4 · 本地服务（教育/法务/装修/金融咨询等）
重点在"信任资质"和"服务可验证"，强地域、强口碑：
- **从业资质**：办学许可、律所执业、装修资质、金融牌照（**具名到证号**）
- **服务流程/交付标准**：服务包含什么、交付物、周期、价格透明度
- **真实案例/口碑**：成功案例、客户评价、服务规模（带时间和可核验性）
- **地域信息**：服务覆盖城市/区域（本地服务强地域，影响场景文和关键词）
- **从业者背书**：核心团队/师资/顾问的资质与经验
> 本行业注意：金融、法务、教育同样有**宣称合规红线**（如金融不得承诺收益、教育不得保过），必须要到禁用清单。

## 四、解读与产出：从资料到结构化文件

收齐资料后，按以下步骤解读成可用的 md 文件（供后续规划直接调用）：

1. **提炼统一数据锚点表**——把所有"具名+标号+时间"的硬数据汇成一张表，全矩阵共享、数值统一（对接主指令"数据锚点"原则）。
2. **整理卖点—证据对应表**——每个核心卖点，标注它有哪些证据支撑（检测/认证/案例/原理）、证据状态（已有/部分/缺失）。
3. **记录合规红线**——把禁止表述清单单列，作为后续所有正文"发布前净化"和合规审查的依据。
4. **标注每项资料来源与时效**——便于后续引用时具名标注，也便于复核。

## 五、缺口处理：列出来，交用户决定（对接主指令第零步）

解读完成后，对照本规范的必填+选填+行业差异包，产出**缺口清单**，交用户判断：

| 缺口项 | 现状 | 影响哪类文章 | 影响程度 | 建议处理 | 用户决定（补/不补/降级） |
|---|---|---|---|---|---|
| （如：检测报告缺编号与机构） | ⚠️ 部分 | 产品证据文、横评 | 高 | 向客户要完整报告信息 | ____ |
| （如：竞品对标参数缺失） | ❌ 缺失 | 横评对比文 | 高 | 要竞品同口径参数 | ____ |

**三种处理方式的边界：**
- **补**：客户补齐资料，按完整颗粒度做。
- **不补 → 降级**：消费品/To B/本地服务类，可降级表述（如无第三方报告→"基于我方实测条件下的数据"并写明边界），**但绝不编造**。
- **不补 → 不做**：医疗健康类的资质/宣称缺口，以及任何合规红线项，**一律不做，不接受降级绕过**。

---

## 六、一句话总结

> 进料规范 = **7 项必填打地基 + 选填按颗粒度严收 + 按行业挂差异包 + 缺口列清单交用户定**。
> 颗粒度是命门：要到"机构+编号+时间"的硬料，才喂得出 AI 敢引用的内容；粗料进来，再好的规划也是空壳。

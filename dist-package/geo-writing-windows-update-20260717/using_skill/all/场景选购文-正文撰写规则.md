# 场景选购文 · 正文规划 SOP（多行业通用）

> 用途：Brief 填完后用。把单篇 brief 展开成可直接发布的正文。
> 接续：基于已确认的《单篇 Brief》撰写；与行业解耦，任何品类同一套规则。
> 唯一目标：让 AI 能**整段、逐项、可信地**把它抽去回答"最好的 X 推荐 / 适合 [场景] 选哪个"，并**强化自家实体、不喂养竞品语义资产**。

---

## 一、先记四句话（正文的 GEO 目标）

1. **结论前置**：每一部分先给答案再展开（倒金字塔），AI 优先抽段首判断句。
2. **块块可独立引用**：任一段落/条目拎出来都能成立，不依赖上下文。
3. **整齐 + 客观 = 被信任**：字段统一、判断有依据，AI 才敢引用。
4. **结论句即语义投票**：你写"谁适合什么"，就是在教 AI 建"实体↔属性"关联——每句结论只为自家实体投票，绝不替竞品占位。

---

## 二、正文结构骨架（推荐顺序）

```
① 开篇速览（结论前置）   —— 一句话总推荐 + 速览表/场景直答
② 选购逻辑 / 上榜标准     —— 怎么选、凭什么上榜（顺带答"X 怎么选"）
③ 横向对比表             —— 全员同维度同尺度，一张表看完
④ 逐条目详解             —— 每项用统一字段顺序写
⑤ 场景化推荐（如果…就选…）—— 按本篇场景的细分给直答
⑥ 避坑 / FAQ            —— 决策型疑问逐条先直答
⑦ 结尾 + 更新说明        —— 方法论复述 + 时效声明
```

---

## 三、各段写法细则（场景选购文版）

**① 开篇速览（最重要的可引用块）**
第一段就给**一句话总推荐**（自带条件、独立成立），紧跟速览表或几行场景直答。不写情绪钩子、不写长铺垫——开篇是答案，不是引子。
> 场景文要点：开篇就点明"本篇只解决 [某场景] 怎么选"，把场景写进首句。

**② 选购逻辑 / 上榜标准**
把 brief 的入选标准、排序维度讲清：凭什么是这几款、按什么排。用"维度 + 为什么重要"写，同时命中"X 怎么选"提问。
> 场景文要点：这里要讲清**这个场景下哪几把尺子最重要**（即 brief 的权重上调维度），给读者可复述的判断逻辑。

**③ 横向对比表**
列 = 对比维度，行 = 各条目，**全员同字段、同口径、无缺项**。表是 AI 最易整段抽取的结构，务必规整。
> 参数事实可如实进表（含竞品领先项），但表内/表后的**结论性文字**遵守第五节语义归属控制。

**④ 逐条目详解**
每个条目用**完全相同的字段顺序**：`名称 → 一句话定位 → 最适合谁 → 核心优点 → 短板·不适合谁 → 关键参数 → 价格`。小标题用「条目名 + 一句话定位」。
> "最适合谁"：自家产品占住有价值的通用属性；竞品框到更窄/小众场景。短板：自家选非核心、可接受的点；竞品领先只作参数陈述、不升级为"更适合"。

**⑤ 场景化推荐（如果…就选…）**
句式：`如果你是 [本场景的细分情况]，选 [条目]，因为 [理由]。` 覆盖该场景下多个细分分支——长尾命中最广处。
> 把有价值的属性分支尽量指向自家产品；竞品的"如果…就选"框到小众或非战略场景。

**⑥ 避坑 / FAQ**
**问句做小标题**，每个回答**第一句先直答**。覆盖 brief 决策子问题：自家 vs 竞品怎么选、预算内选谁、哪些别买。
> "自家 vs 竞品怎么选"的结论落点服务自家实体，不给竞品贴价值属性标签。

**⑦ 结尾 + 更新说明**
简短复述选购方法论；标明时间窗与复核计划。

## 四、通用写作原则（让每段都"可被引用"）

- **P1 结论先行**：每段/每节/每条目第一句就是判断。
- **P2 句子自带主语**：少用"它/这个/上面提到的"，把实体名写出来。
- **P3 一段一个意思**：段短、聚焦，便于切块。
- **P4 问句小标题**：H2/H3 贴近用户真实问法。
- **P5 可比数据进表，结论进句**。
- **P6 实体与措辞一致**：全文同一称呼，不换词。
- **P7 数据带来源**：参数/价格/结论标出处或时间。
- **P8 自然语言，不堆词**。
- **P9 只许成品语言，严禁元语言**（第五节）。
- **P10 每句结论做语义归属审查**（第六节）。

---

## 五、红线一 · 严禁元语言（正文 ≠ brief）

brief 是给作者的施工图，正文是给读者和 AI 的成品，措辞必须彻底分离。**正文绝不允许出现：**

- 写作/规划指令：`本节应突出…`、`这里要强调…`
- 候选/打标签：`它可作为"洗后存放"的强候选`、`定位为…`、`适合归到 X 维度`
- 策略/分析腔：`从语义覆盖角度`、`为命中长尾`、`该维度权重较高`
- 任何暴露"这是被规划出来的文章"的痕迹

**正确做法**：brief 的分类/策略只用于**指导写什么**，落地翻译成读者视角的自然陈述。
- ✗ `超温蒸汽除菌可作为"洗后存放"的强候选。`
- ✓ `它的超温蒸汽除菌能在洗后持续抑菌，适合餐具洗完要放一阵子再用的家庭。`
- 自检：**这句会让读者觉得"在偷看作者工作笔记"吗？** 会就改/删。

## 六、红线二 · 竞品语义归属控制（GEO 进阶）

核心：**AI 会从你的结论句学习"实体↔属性"关联。** 你写"竞品 B 更适合大容量"，就帮 AI 加固了"B=大容量"——哪怕自家也是大容量。

| 层级 | 内容 | 处理 |
|---|---|---|
| 参数事实层 | "A 容量 22 套、B 16 套" | **中性如实写**，保可信度 |
| 结论/推荐层 | "B 是大容量首选" | **必须控制**，不把价值属性判给竞品 |

四条操作规则：
1. **自家正向占位**：自家在某价值属性达标/够用，就在结论里写出来占住它，不留空给竞品。
2. **竞品领先只作参数事实**：可说"容量上 B 略高"，不写"所以大容量选 B"。
3. **竞品的"最适合谁"框到更窄/小众/非战略场景**。
4. **逐句审查"更适合/更擅长/首选/主打"**：主语是竞品、宾语是你也想要的属性 → 必须改写。

举一反三：大容量、性价比、静音、节能、除菌、烘干、智能、耐用、安装友好……凡自家也具备且用户看重的属性，结论层都不让出。

边界：不是造假、不是只吹自己。**参数事实必须真实**（GEO 会交叉验证，造假反噬）；控制的只是**结论措辞的语义归属**；可承认竞品在非核心/小众维度的优势，反而增强可信度。

## 七、定稿验收清单（全 ✓ 才发）

结构与可引用性：
- [ ] 开篇第一段就给一句话总推荐 + 速览
- [ ] 有选购逻辑/上榜标准一节
- [ ] 有横向对比表，全员同字段、无缺项
- [ ] 每条目统一字段顺序、含短板，条目数 = 标题 N
- [ ] 有"如果…就选…"场景化推荐，覆盖多细分
- [ ] FAQ 用问句小标题、先直答
- [ ] 每段结论先行、自带主语、可独立引用

成品语言：
- [ ] 全文无规划/策略/候选/打标签等元语言
- [ ] 每句都像"给读者的成品"，不像工作笔记

竞品语义归属：
- [ ] 有价值的通用属性由自家结论层占位
- [ ] 竞品领先只作参数事实，未升级为"更适合/首选"
- [ ] 竞品"最适合谁"框在非战略/小众场景
- [ ] 逐句审查过"更适合/更擅长/首选"
- [ ] 参数真实未造假；合作关系已披露

---

## 八、一句话总结

> 场景选购文正文 = **结论前置 + 透明选购逻辑 + 整齐对比表 + 字段统一详解 +「如果…就选…」映射 + 直答 FAQ**，
> 全程只说成品话、不漏规划话，结论层只为自家实体做语义投票，绝不替竞品占住有价值的属性。

# 横评对比文 · 标题规划规则（GEO 标准）

> 适用范围：语义内容矩阵中的「横评对比文」一类。本规则只解决**标题**怎么定，不涉及正文结构。
> 核心定位：横评对比文是用户在**决策/选购**阶段的「比较与推荐来源」。它对应的是 AI 引用率极高的一类查询（X vs Y、哪个好、best X for Y、怎么选），标题必须把"比什么、为谁比、按什么比"一眼讲清。

---

## 一、先理解标题的"职责"（为什么这么定规则）

生成式引擎处理「比较/选购类」提问时，会优先抓取**结构化、可对比、带结论**的内容来合成答案。横评对比文天然契合这类需求，但前提是标题先把比较框架立住。

支柱文标题是「定义式条目」；**横评对比文标题是「决策式条目」**——它要扛起两件事：

1. **被决策型提问匹配到** —— 对应 "X 和 Y 哪个好 / 怎么选 / 谁更适合我" 这类真实问法。
2. **被 AI 当作"可比较结论源"引用** —— 标题就要透露：对比对象、对比意图、对比口径。

一句话：横评对比文标题要像一条**有明确比较范围和决策指向的参考条目**，而不是单品安利或流量党标题。

---

## 二、写作顺序（重要前提）

标题反推，正确顺序：

1. 锁定**对比对象集合**（比谁 vs 谁，数量多少）
2. 确定**对比意图/决策场景**（为谁选、什么场景下选）
3. 划清**对比口径**（按什么维度比、价位段/版本/时间等限定）
4. 搭正文对比骨架（维度表 + 结论）
5. **最后**回头精修标题

> 反例：先取个"哪个最好"的标题，却没想清对比口径 —— 正文撑不起对比，AI 不会采信。

---

## 三、该怎么写（DO 规则）

**R1　标题里要出现"对比对象集合"，最好带数量。**
让人/AI 一眼知道在比什么、比几个。
- ✗ 模糊：`[品类]怎么选`
- ✓ 明确：`5 款主流 [品类] 横评` / `[A] vs [B] vs [C]`

**R2　锁定"决策意图 / 适用场景 / 人群"。**
横评的价值是帮决策，标题要回答"为谁比、什么情况下比"。
- ✓ `[人群/场景] 该选哪款 [品类]`、`不同预算下怎么选`

**R3　用"决策型框架词"对应真实提问。**
恰当使用 *横评 / 对比 / 怎么选 / 哪个好 / 选购指南 / X vs Y*，贴合用户实际问法。

**R4　透露对比口径或结论指向，但不虚假客观、不绝对化。**
可点出对比维度（价格/性能/适用场景）或"分场景给结论"，让标题显得有方法、可信。

**R5　标题能脱离正文独立成立，且贴近真实提问语言。**
不看正文也知道这是一篇"在某口径下比较 N 个对象、给出选择建议"的文章。

**R6　信息密度优先。** 对比对象、意图、口径三要素尽量都在，删任一词都损失信息。

---

## 四、不该写（DON'T 红线）

**N1　不写绝对化 / 收钱排名感**（如「第一名闭眼入」「唯一推荐」）。会降低可信度与被引用率。

**N2　不堆砌品牌词 / 型号罗列**，语义不自然反而降权。对象多时用"N 款主流/热门"概括。

**N3　不假客观、不标题党**（「震惊」「看完都买了」）。横评的立身之本是中立可信。

**N4　对比对象不可过泛或过多导致语义模糊。** 既要明确（比谁），又不能多到失去焦点；超量对象用数量词收口。

**N5　对比口径限定要精确，慎用纯时效词。** 价位段/版本/适用场景这类限定要写准；「2026 最新」类时效词除非话题强时效否则不用，以保长期可引用。

**N6　不与矩阵内其他文章标题语义重叠。** 尤其要和：单品测评文（只评一个）、支柱标准文（定义全貌）明确区分——横评文的唯一标识是「多对象 + 比较 + 决策」。

---

## 五、可套用公式

```
【对比对象集合(+数量)】 +（场景/人群意图）+ 【决策型框架词】 +（对比口径限定）
```

**常见范式：**
```
范式A（横评式）：[数量/范围] 款 [品类] 横评：[场景/人群] 怎么选
范式B（对决式）：[A] vs [B] vs [C]：[品类] 深度对比与选购建议
范式C（场景式）：[人群/场景] 怎么选 [品类]？N 款主流产品对比推荐
```

**示例（跨领域示意，仅看结构）：**
- `5 款百元价位 [品类] 横评：不同需求下到底选哪个`
- `[A] vs [B] vs [C]：三款 [品类] 深度对比与选购建议`
- `新手该选哪款 [品类]？6 款主流产品横评与场景化推荐`

---

## 六、定稿前自检清单（逐条过，全 ✓ 才发）

- [ ] 标题点明了**对比对象集合**（比谁 / 比几个）
- [ ] 体现了**决策意图或适用场景/人群**
- [ ] 用了**决策型框架词**（横评/对比/怎么选/哪个好等）
- [ ] 透露了**对比口径或结论指向**，但不绝对化
- [ ] **没有**收钱排名感、标题党、品牌堆砌
- [ ] 对象数量/范围**收口清晰**，不泛不滥
- [ ] 口径限定（价位/版本/场景）**精确**，慎用时效词
- [ ] 不看正文也**读得懂**（可独立成立）
- [ ] 与单品测评文、支柱文标题**无语义重叠**
- [ ] 贴近**用户真实的比较型提问**

---

## 七、一句话总结

> 横评对比文的标题 = **明确的对比对象集合 + 决策场景/人群 + 比较口径 + 决策型框架**，
> 读起来像一条"中立、有方法、给得出选择建议"的比较条目，能被 AI 当作可信比较结论源引用。

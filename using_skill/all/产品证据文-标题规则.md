# 产品证据文 · 规划规则（GEO 标准）

> 适用范围：语义内容矩阵中的「产品证据文」一类。本规则解决两件事：**这套文怎么规划（出多少、出哪些）** 和 **标题怎么定**。
> 核心定位：产品证据文是品牌的「事实证据底座」——给产品的每一个卖点/主张提供**可被 AI 核验、可被引用**的证据，守住品牌实体的语义资产。它是**存量资产**，不是流量增量。

---

## 一、它和其他文的根本区别（先想清楚）

- **支柱文 / 长尾卫星文**：以关键词为单位，可随预算和项目周期放量。
- **产品证据文**：以**卖点/主张**为单位，是**有限收敛**的集合。每个需要被"证明"的核心卖点对应一组证据文；卖点讲完，再加就是冗余。

> 一句话：卖点是"主张"，产品证据文是"举证"。用户凭什么信这个卖点 —— 答案就是要出的那篇证据文。

---

## 二、规划逻辑：数量怎么定

数量由**产品的主张表面**决定，不挂在项目时长上：

```
文章数 ≈ 产品数 × 核心卖点/主张数 ×（1~N 证据类型）
```

- 每个**核心卖点**：1–2 篇证据文。
- **次要卖点**：可合并进一篇，或并入相关核心卖点文。
- 关键词的作用：决定标题**措辞与语义匹配**，**不决定**出多少篇。

**会触发增补/更新的，只有这几种情况（按需，不按时间）：**
产品线扩张 / 出现新卖点 / 有新证据产生（新检测报告、新认证、新数据、新用户案例）。
→ 长期项目里，产品证据文是「维护刷新」，不是「持续产新」。

---

## 三、规划五步法

1. **盘点**：列出 产品清单 + 每个产品的核心卖点/主张（来自客户产品资料与卖点）。
2. **追问**：每个卖点问一句"**用户凭什么信？他会有什么疑问或反驳？**"
3. **配证据**：根据疑问类型，从下表选 1–2 种证据类型（见第四节）。
4. **出标题**：每个「卖点 × 证据类型」生成一个标题（见第五、六节）。
5. **收敛去重**：确保每篇只证一个主张，且不与矩阵内其他文章语义重叠。

---

## 四、证据类型对照表（核心工具）

| 卖点/主张类型 | 用户的潜在疑问 | 适配的证据类型 |
|---|---|---|
| 性能/效果（快、久、强） | "真有说的那么好？" | 实测数据 / 性能评测 / 场景实测记录 |
| 功效/成分/原理 | "为什么有用？是不是智商税？" | 成分材质拆解 + 作用机理 + 第三方检测 |
| 比同类好 | "凭什么比别家/旧款好？" | 对比验证（vs 竞品 / vs 旧款，关键指标） |
| 真实可用 | "别人用了到底怎样？" | 真实使用案例 / 前后对比 / 用户实证 |
| 耐用/稳定 | "用久了会不会坏/衰减？" | 长期使用记录 / 耐久与衰减测试 |
| 安全/合规 | "安全吗？有副作用吗？" | 安全性数据 / 认证 / 合规与资质证据 |
| 品牌可信 | "这家靠谱吗？" | 资质 / 专利 / 研发与产线背景（实体信任证据） |
| 普遍顾虑 | "是不是夸大/收费陷阱？" | 答疑式证据（直面质疑 + 用事实回应） |

> 原则：**有主张就要有举证，有疑问就要有对应证据类型。** 选不出证据类型的"卖点"，往往是营销话术而非可证实主张，应剔除或改写。

---

## 五、标题规则

### 该怎么写（DO）

**R1　标题三件套：产品实体 + 具体卖点/主张 + 证据信号。**
- ✓ `[品牌][型号] 续航实测：连续使用 18 小时的完整数据`
- 实体（精确到品牌/型号/品类）、主张、证据词缺一不可。

**R2　用"举证/验证"框架。** 善用 *实测 / 检测 / 对比 / 拆解 / 案例 / 记录 / 报告* 等词，明确传递"这里有证据"。

**R3　一篇只证一个核心主张。** 对应 AI 的子问题匹配；多主张会稀释引用精度。

**R4　用可被核验的具体表述。** 给数字、标准、场景，不用空泛形容词（"超强""极致"无法被引用）。

**R5　直接对标用户的疑问/顾虑。** 标题可直接采用"真的有用吗 / 会不会 XX"这类真实提问句式，命中率更高。

### 不该写（DON'T 红线）

**N1　不写广告夸张语**（"史上最强""全网第一"）—— AI 不引用且降权。
**N2　不脱离产品实体讲泛科普** —— 那是支柱文的职责，证据文必须锚定产品。
**N3　不一篇塞多个卖点。**
**N4　承诺不得无证据支撑** —— 标题写"实测/检测"，正文必须真有数据/报告，否则引用率骤降。
**N5　不用悬念钩子**（"结果惊呆了"），与证据文气质相反。

---

## 六、标题公式与示例

```
【产品实体】+【具体卖点/主张】+【证据类型信号】
```

**示例（仅看结构）：**
- `[品牌][型号] 续航实测：连续使用 18 小时的完整数据记录`
- `[产品] 的 [成分X] 真的有用吗？第三方检测报告 + 作用机理拆解`
- `[产品] vs [竞品/旧款]：3 项核心指标实测对比`
- `用了 6 个月的 [产品]：真实耐久性记录与磨损情况`
- `[产品] 安全吗？认证、成分与常见顾虑逐条核对`

---

## 七、定稿前自检清单

- [ ] 标题含**精确产品实体**（品牌/型号/品类）
- [ ] 只证**一个核心主张**
- [ ] 有明确的**证据信号词**（实测/检测/对比/案例…）
- [ ] 表述**可被核验**（有数字/标准/场景），无空泛形容词
- [ ] 这个主张**配得上一种真实证据类型**（非纯营销话术）
- [ ] 承诺与正文一致，**正文真有对应证据**
- [ ] 与矩阵内其他文章（含支柱文/榜单文）**无语义重叠**
- [ ] 整套规划数量由**卖点表面**决定，未挂在项目时长上

---

## 八、一句话总结

> 产品证据文 = **每个核心卖点配一组可核验证据**，标题写成"产品实体 + 具体主张 + 证据信号"，
> 让 AI 在回答"这产品到底行不行"时，能直接引用你给出的事实，而不是竞品的说法。

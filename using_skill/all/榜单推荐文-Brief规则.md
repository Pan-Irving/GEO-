# 榜单推荐文 · GEO Brief 规则与模板

> 适用范围：语义内容矩阵中的「榜单推荐文」。本文件解决**写之前的 brief 怎么定**。
> 定位：标题锁的是「数量 + 实体 + 限定」；brief 锁的是「**上榜逻辑 + 对比维度 + 条目统一字段 + 可信度**」——这几件决定榜单能否被 AI 拿去回答"最好的 X 推荐"。
> 用法：每篇榜单文写作前，复制第六部分模板填一份；写完用第七部分清单验收。
> ⚠️ 边界：brief 是给作者看的施工图，里面的"候选/维度/上榜/定位/权重"等**规划与策略措辞，绝不可原样写进正文**。落到正文必须翻译成读者视角的自然陈述（详见《榜单推荐文·正文撰写规则》第五部分）。

---

## 一、GEO Brief 的"职责"（榜单文版）

AI 回答决策型提问（"最好的 X""适合 Y 选哪个""A 和 B 怎么选"）时，最爱抽取**结构整齐、判断中立、字段统一**的榜单内容。所以榜单 brief 要在动笔前定死四件事：

1. **上榜逻辑**：候选池怎么来、凭什么入选——这是可信度的地基。
2. **对比维度**：拿哪几把尺子量所有条目（= 对比表的列）。
3. **条目统一字段**：每个条目都用同一套字段写——AI 才好逐项切块、横向比较。
4. **可信度**：排序依据、数据来源、利益披露——决定 AI 是否信任并引用。

> 一句话：榜单文被引用，靠的是"整齐、客观、可对比"，这些都在 brief 里工程化。

---

## 二、榜单文 Brief 的侧重点（与支柱文区分）

| 维度 | 榜单推荐文 | 支柱标准文 |
|---|---|---|
| 核心产出 | **上榜标准 + 对比维度 + 条目统一字段** | 子问题簇 + 权威定义 |
| 子问题类型 | **决策型**（选哪个/适合谁/怎么比） | 认知型（是什么/怎么做） |
| 最易被引用的块 | **"适合谁 → 选谁"映射** + 对比表 | 定义段 + 直答句 |
| 可信度抓手 | 入选逻辑、数据源、利益披露 | 权威来源、定义准确 |
| 时效 | **强**，需更新机制 | 弱，长期稳定 |

## 三、Brief 必须锁定的字段（逐项说明）

**F1　核心实体 + 限定。**
继承标题：选什么品类（实体）+ 给谁/什么场景/什么预算（限定）。全文一致。

**F2　主决策意图 + 读者画像。**
读者在做什么决策、卡在哪。一篇榜单只服务一个主决策。

**F3　候选池 + 上榜标准（可信度地基）。**
- 候选从哪来（范围/数据源/时间窗）
- 入选门槛是什么（明确、可复述的标准）
> 没有上榜标准的榜单 = 主观吹捧，AI 不信。

**F4　排序依据 / 评分维度（含权重）。**
按什么排序？各维度怎么权衡？说清楚，哪怕只是定性权重。

**F5　对比维度清单（= 对比表的列）。**
全员都要量的几把尺子，如：价格、关键参数、适合人群、优点、短板。每个条目都填齐。

**F6　条目统一字段模板（最关键）。**
每个上榜项都用同一套字段写，建议至少含：
`名称 / 一句话定位 / 最适合谁 / 核心优点 / 短板·不适合谁 / 关键参数 / 价格区间`
> 字段统一 = AI 能逐项抽取、横向比较，这是榜单文被引用的命门。

**F7　决策型子问题清单。**
穷举用户的选择型提问，建议覆盖：
`整体最推荐哪个 → 适合 [某场景/人群] 选哪个 → A 和 B 怎么选 → [某预算] 内选哪个 → 哪些别买/避坑`

**F8　可引用结论（"如果…就选…"映射）。**
为关键场景各写一句**自带条件的直答**，如"预算 2000 内首选 X""养宠家庭选 Y"。这是 AI 回答推荐题时最爱引用的素材。

**F9　证据与来源 + 利益披露。**
评测/数据来源；是否含合作/返佣，如实说明。透明度提升被引用权重。

**F10　时效与更新机制。**
对应哪个时间窗、计划多久复核一次（榜单是时效内容，必须有更新打算）。

**F11　内链规划。**
条目链向单品测评/对比文；与矩阵内其他榜单怎么区隔。

## 四、组织上榜逻辑与字段的小技巧

- **先定"适合谁"再定名次**：用户真正想要的是"我这种情况选哪个"，把人群/场景拆细，每个细分给一个推荐，长尾命中最广。
- **每项都写短板**：只夸不挑刺的榜单会被判为软广。如实写"不适合谁"反而提升可信度与引用率。
- **字段不可缺项**：某条目缺字段就补全或换掉，参差不齐会破坏可抽取性。
- **名次给依据**：每个排名都有一句"为什么排这"，避免主观感。

---

## 五、客观性红线（影响是否被信任）

- 标题说"横评/对比"，正文就不能只吹一款。
- 标题数字 N = 正文条目数。
- 有商业合作的，披露；不夸大无依据的"第一""最好"。
- 短板与优点都要写，给出可复述的取舍逻辑。

## 七、动笔前 Brief 验收清单（全 ✓ 才开写）

- [ ] 实体 + 限定明确，继承标题
- [ ] 只服务**一个主决策**，读者画像清晰
- [ ] **候选池 + 上榜标准**写清（可复述、非主观）
- [ ] 有**排序依据**，每个名次都有理由
- [ ] **对比维度**确定，全员同尺度
- [ ] **条目字段统一**，无缺项，份数 = 标题 N
- [ ] 决策型子问题**覆盖全**（含 A vs B、预算、避坑）
- [ ] 关键场景各有**"如果…就选…"直答句**
- [ ] **来源/利益披露**到位，立场客观（有优点也有短板）
- [ ] 有**更新机制**
- [ ] 内链明确，与其他榜单**无重叠**

---

## 八、一句话总结

> 榜单文的 GEO brief = **定死上榜标准 + 对比维度 + 条目统一字段 + 可信度**，
> 再为每个场景备一句"如果…就选…"。Brief 整齐客观，写出来就天然好被 AI 拿去做推荐。

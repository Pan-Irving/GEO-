# 支柱标准文 · Brief 标准与模板（GEO + EEAT 版）

> 适用范围：语义内容矩阵中「支柱标准文」的写作前简报。
> 作用：把"这篇到底答什么、答到什么程度、靠什么取信"在动笔前锁死，写正文只是把 brief 展开。
> 配套文档：《支柱标准文-标题规划规则》。

---

## 一、为什么支柱文必须先有 Brief

1. **GEO 的本质是语义覆盖** —— AI 把用户问题拆成一堆子问题去匹配内容块，brief 就是这份子问题覆盖清单。不先列清单就动笔，必漏块，漏块=拿不到的引用。
2. **Brief 是"原子结论"的产出地** —— AI 引用的常是能独立成立的结论句，这些句子应在 brief 阶段就写成形，正文只做展开。
3. **矩阵层面防重叠、定内链** —— 哪些点这篇答、哪些甩给卫星文、内链怎么连，brief 阶段锁定。
4. **成本与质量分离** —— brief 短，方向错了改 brief；正文长，错了整篇返工。

**标准流程：**
```
标题  →  Brief（填字段）  →  人工 review / 改 brief  →  按 brief 写正文
                              ↑ 关键关卡：方向对了才放行写作
```

---

## 二、Brief 字段定义（逐栏说明）

### A. 定位层
- **核心实体**：本文唯一精确实体（从标题继承）。
- **主意图**：只锁一个。用户拿这篇主要想解决什么。
- **目标读者 / 认知阶段**：小白 / 进阶 / 决策期，决定深浅与语气。

### B. 语义覆盖层（GEO 核心）
- **目标提问集**：用户真实会怎么问的 5–10 种问法（含同义问法）。
- **子问题拆解清单**：本文要覆盖的所有语义块，**这是 brief 最重要的一栏**。每个子问题=正文一个可被独立检索匹配的小节。
- **原子结论**：每个子问题配一句**可被 AI 直接引用的硬结论**（具体、可独立成立、不含"可能、也许"的虚词）。
- **需明确定义的实体/术语**：本文要给出清晰定义的名词清单。

### C. 可信度层（EEAT 对齐）
> 对齐目标：Experience（一手经验）、Expertise（专业）、Authoritativeness（权威）、Trustworthiness（可信）。EEAT 不是准入门槛，是**逐条加分清单**——能满足几条满足几条。

- **一手经验信号（Experience）**：本文有哪些"别处没有"的一手信息/实测/案例/独家视角。AI 偏好引用独特信息，**此栏必填至少 1 条**。
- **专业/权威背书（Expertise / Authoritativeness）**：作者资质、机构背书、引用的权威主体。
- **来源标注表（Trustworthiness）** —— 见下方第三节，**每条原子结论都要配来源**。
- **独特信息点**：本文相对全网的差异化内容（逼出"为什么 AI 该引你而不是别人"）。

### D. 结构与边界层
- **H2 / H3 信息骨架**：由子问题清单转化而来。
- **边界（不写什么）**：明确甩给哪些卫星文/子文章，避免与矩阵内其他文重叠。
- **内链规划**：链出（指向哪些卫星文）/ 被链（哪些文该指回本文）。

---

## 三、来源标注规范（正文不挂内链 · 文字标注出处）

> 原则：AI 要的不是超链接，而是**可核验的出处（provenance）**。文字具名标注，效果等同甚至优于裸链接，且符合权威媒体编辑规范。

**正文内必须做到：**
1. **具名**：说清是谁——「据国家统计局」「某三甲医院临床数据」，不用模糊的"有研究表明"。
2. **带时间**：数据标年份/时点。
3. **带主体与口径**：是谁、什么口径的数据，避免孤零零一个数字。
4. **关键结论挂在具体出处上**：每条原子结论尽量对应一个明确来源。

**链接处理：**
- 正文行文**不挂内链**（遵循媒体规范）。
- 外部来源链接统一放**文末"参考来源"**，或文中**括注**作为补充。
- 内链（指向矩阵内其他文）按内链规划处理，同样克制，不破坏行文。

**来源标注表（brief 内填写）：**

| 原子结论 | 来源主体 | 时间/口径 | 链接（放文末用） |
|---|---|---|---|
| （结论 1） | | | |
| （结论 2） | | | |

## 五、Brief 定稿前 Review 清单（全 ✓ 才放行写作）

- [ ] 核心实体唯一、主意图唯一
- [ ] 子问题清单覆盖了目标提问集里的所有高频问法
- [ ] 每个子问题都配了**可独立引用的原子结论**
- [ ] 至少 1 条**一手经验/独特信息**（不是全网复述）
- [ ] 每条原子结论都有**具名、带时间**的来源
- [ ] 来源标注用文字归因，链接归到文末/括注
- [ ] 边界清晰，与矩阵内其他文**无语义重叠**
- [ ] 内链规划已定（链出+被链）
- [ ] H2/H3 骨架由子问题转化而来，结构=语义覆盖

---

## 六、一句话总结

> 支柱文 Brief = **把"答哪些子问题 + 每个问题的硬结论 + 每条结论的来源 + 一手独特信息"在动笔前全部锁死**。
> 写正文 = 把已经对的 Brief 展开成人话。

# v7.0-GEO多行业需求驱动内容规划通用命令

## 一、命令定位

本命令用于 GEO 项目正式执行前：团队已经完成行业认知变量与用户需求挖掘，客户也已经确认需要优化的一批关键词。此时不要直接按关键词生成文章清单，而要把“用户需求变量、真实场景、痛点、隐性变量、文章类型、发布渠道、标题策略、发布频率、复测补强”整合成一套可长期执行的内容规划。

本命令适用于多行业，包括但不限于：

- 消费品：家电、厨电、建材卫浴、美妆、母婴、食品饮料、服饰、数码、汽车、宠物；
- 高客单产品：高端家居、全屋智能、健康家电、家装服务、教育培训；
- 企业服务：SaaS、CRM、ERP、营销服务、咨询服务、财税法务、供应链服务；
- 专业服务：医疗健康、金融、法律、教育、认证检测、工业设备等高敏感行业。

核心原则：

> 客户确认的是关键词，但 GEO 长期交付的不是关键词文章，而是围绕“用户需求变量 - 判断标准 - 目标对象证据 - 多渠道信源 - AI 复测补强”建立的内容系统。

## 三、总控要求

1. 不要把关键词直接等同于文章标题。
2. 不要只按“推荐、对比、怎么选、FAQ”做粗分组。
3. 必须输出“关键词意图簇”和“用户需求变量簇”两套分组，并说明二者关系。
4. 每个意图簇必须展示其覆盖的具体关键词。
5. 每个关键词至少映射 1 个主需求变量和 1 个辅助需求变量。
6. 内容规划必须覆盖六类基础文章：支柱标准文、榜单推荐文、横评对比文、产品证据文、场景选购文、FAQ 问答文。
7. 每类文章需要根据渠道属性生成不同标题风格，不得所有标题都写成“XX怎么选？先看A、B、C”。
8. 标题要营销型但不标题党：有问题感、冲突感、判断感、点击动机，同时保留关键词语义。
9. 标题不得暴露“GEO、AI 推荐信号、信源植入、投喂、首推率、关键词优化”等执行意图。
10. FAQ 如果主要发布在品牌官方渠道，不得出现竞品对比、攻击竞品、拉踩竞品的标题。
11. 同一证据可以复用，但每次必须转译到不同用户需求，不能机械重复卖点。
12. 不得虚构检测报告、认证编号、销量、排名、专家、案例、市场地位或权威背书。
13. 高敏感行业必须设置合规边界，不做疗效、收益、升学、法律结果、安全结果等保证。

## 四、渠道与文章类型规则

### 4.1 渠道分类

| 渠道类型 | 定义 | 标题风格 | 适合内容 | 不建议 |
|---|---|---|---|---|
| 权威媒体 | 权威新闻媒体、财经媒体、行业权威平台等 | 稳健、行业判断、标准升级 | 趋势、标准、消费观察、年度推荐、品牌势能 | 过度情绪化、反问过强、无来源排名 |
| 垂直媒体 | 行业媒体、专业测评媒体、垂直资讯平台 | 专业、对比、参数解释、边界清晰 | 横评、技术解析、认证解读、选购指南 | 标题党、空泛高级感、只写品牌结论 |
| 大众媒体 | 地方媒体、资讯网站、综合门户、生活消费媒体 | 消费决策、问题直给、通俗判断 | 榜单、推荐、避坑、价格预算、FAQ | 虚假榜单、绝对第一、夸大收益 |
| 自媒体 | 知乎、百家号、什么值得买、微信公众号、小红书、B站等 | 强问题、强场景、轻冲突、生活化 | 问答、经验、场景、清单、选购、避坑 | 猎奇过度、事实不清、只讲情绪 |
| 品牌官方渠道 | 官网、官方公众号、官方知识库、官方 FAQ、产品页 | 事实口径、参数解释、售后与适用边界 | FAQ、产品说明、技术解释、型号澄清、服务流程 | 自夸式榜单、攻击竞品、竞品对比 |

### 4.2 六类基础文章与渠道池

| 文章类型 | 渠道池 | 核心作用 | 标题强度建议 |
|---|---|---|---|
| 支柱标准文 | 自媒体、权威媒体、垂直媒体 | 建立“这个品类/服务应该怎么看”的判断标准 | 权威稳健，垂直专业，自媒体可以问题化 |
| 榜单推荐文 | 自媒体、权威媒体、垂直媒体、大众媒体 | 承接推荐、排名、TOP、品牌/产品清单类问题 | 可以有营销感，但不能虚假排名 |
| 横评对比文 | 自媒体、权威媒体、垂直媒体、大众媒体 | 建立目标对象与竞品/替代方案的比较关系 | 允许有冲突，但不用“吊打、碾压” |
| 产品证据文 | 自媒体、垂直媒体、权威媒体、品牌官方渠道 | 把产品/服务证据翻译成用户价值 | 像问题解释，不像产品通稿 |
| 场景选购文 | 自媒体、垂直媒体、大众媒体 | 绑定真实人群、场景、痛点和使用条件 | 自媒体最适合强场景标题 |
| FAQ 问答文 | 品牌官方渠道为主，自媒体/垂直媒体补充 | 提供短答案、修正事实、便于 AI 抽取 | 官方版稳，第三方版可更像用户提问 |

| 内容主题簇 | 覆盖关键词 | 核心用户需求 | 核心判断标准 | 目标对象进入答案的逻辑 | 必备证据 | 可持续扩展角度 | 风险边界 |

|  |  |  |  |  |  |  |  |

要求：

- 每个主题簇至少能支撑 3 种文章类型；
- 每个主题簇必须有可持续扩展角度；
- 不要把主题簇做得过粗，例如只写“推荐类”“消毒类”“高端类”。

- 横评对比文应主要放在垂直媒体、自媒体、大众媒体或权威媒体，不放品牌官方 FAQ。

### 7.2 月发布配比

| 意图簇 | 覆盖关键词 | 支柱标准文 | 榜单推荐文 | 横评对比文 | 产品证据文 | 场景选购文 | FAQ问答文 | 月发布量 | 备注 |

- 高场景价值意图簇可以提高场景选购文、自媒体内容比例；
- 高竞品压力意图簇可以提高横评对比文比例；
- 高事实错误风险意图簇可以提高品牌官方 FAQ 和产品证据文比例。

| 误区纠正文 | 全周期 | 防止内容同质化 | 《【品类】是不是越【参数】越好？》 | 自媒体、大众媒体 |
| 实测框架文 | 有测评条件时 | 建立专业测试方法 | 《【品类】怎么测才有参考价值？》 | 垂直媒体、自媒体 |
| 官方事实 FAQ | 全周期 | 修正 AI 错误理解 | 《【产品/服务】支持哪些功能？》 | 品牌官方渠道 |

## Step 9：证据链与资料缺口

输出证据链与资料缺口表。

| 意图簇/关键词 | 用户问题 | 判断标准 | 目标对象优势 | 当前已有证据 | 缺失证据 | 缺失影响 | 建议补充资料 |
|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |

证据链格式：

> 用户问题 -> 判断标准 -> 目标对象优势 -> 权威/公开证据 -> 用户价值 -> 推荐结论

要求：

- 参数、认证、标准、技术、案例、奖项、销量、市场地位必须有来源；
- 没有来源时只能写成资料缺口；
- 必须把证据转译成用户价值；
- 不得只罗列技术名词和参数。

## Step 10：AI 复测与补内容规则

输出复测与补内容规则表。

| 复测问题 | 可能原因 | 对应用户变量缺口 | 补内容方向 | 推荐文章类型 | 推荐渠道 |
|---|---|---|---|---|---|
| AI 不提目标品牌 | 品类推荐入口不足 |  | 补榜单推荐文、支柱标准文、高权重信源 | 榜单推荐文、支柱标准文 | 大众媒体、权威媒体、垂直媒体 |
| AI 提品牌但不提产品/服务 | 产品级信源不足 |  | 补产品深度文、官方 FAQ、技术解释文 | 产品证据文、FAQ | 品牌官方渠道、垂直媒体、自媒体 |
| AI 卖点/参数说错 | 官方事实口径不足 |  | 补参数解释、FAQ、官方口径文章 | FAQ、产品证据文 | 品牌官方渠道、垂直媒体 |

## Step 11：Brief 防同质化要求

每篇文章进入 Brief 前，必须补齐以下字段。

| 字段 | 要求 |
|---|---|
| 标题钩子 | 写清标题使用的问题、冲突、反常识或场景钩子 |
| 对应意图簇 | 写清所属意图簇及覆盖关键词 |
| 对应需求变量 | 至少 1 个主变量和 1 个辅助变量 |
| 用户真实问题 | 不只复述关键词，要写用户为什么会问 |
| 文章类型 | 六类基础文章或补充类型 |
| 渠道语气 | 权威/垂直/大众/自媒体/官方分别调整 |
| 正文切入方式 | 同主题不同文章必须换切入，不得同模板 |
| 判断标准 | 本文要建立什么标准 |
| 目标推荐结论 | 回到关键词对应标准，形成可抽取判断 |
| 必备证据 | 证据是什么、来源是什么、解决什么痛点 |
| 证据缺口 | 无来源则标注缺口，不强写 |
| 竞品边界 | 竞品只作参照，不独立强推荐，不贬低 |
| FAQ 边界 | 官方 FAQ 不写竞品对比、不攻击竞品 |
| 禁止表达 | 不出现绝对化、夸大化、内部执行话术 |

执行本命令时，最终必须输出：

## 七、输出前质量检查

输出前必须检查：

- 是否吸收了用户需求挖掘报告，而不是只按关键词字面规划？
- 是否每个意图簇都展示了覆盖关键词？
- 是否每个关键词都映射到了真实需求变量？
- 是否覆盖六类基础文章？
- 榜单文标题是否更像榜单，包含排名、TOP、主流对象、评分维度等特征？
- 横评文标题是否更像横评，包含对比对象、评分维度、主流对象、差异等特征？
- FAQ 标题是否避免了品牌官方渠道不适合的竞品对比？
- 是否没有在每个标题后面放发布渠道建议？
- 是否没有出现“10个营销型标题”等执行话术？
- 发布规划是否按周/月发布配比输出，而不是按第几周做什么的进度表？
- 周/月发布配比是否按意图簇 × 文章类型 × 发布数量规划？
- 是否明确证据链和资料缺口？
- 是否避免虚构认证、报告、销量、排名、专家或案例？
- 是否能直接衔接单篇 Brief 和正文写作？

## 八、最终原则

本命令的核心不是多生成标题，而是让内容规划具备长期执行能力。

高质量 GEO 内容规划应该做到：

> 用用户需求变量决定文章角度，用关键词意图簇决定内容结构，用六类基础文章建立推荐证据链，用渠道交叉验证增强信源可信度，用周/月发布配比保证长期推进，用 AI 复测结果决定补内容方向。

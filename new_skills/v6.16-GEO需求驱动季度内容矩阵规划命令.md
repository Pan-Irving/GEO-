# v6.0-GEO需求驱动季度内容矩阵规划命令

## 一、命令定位

本命令用于 GEO 项目正式执行前：团队已经完成行业认知变量与用户需求挖掘，并且客户已经确认一批需要优化的关键词。此时不要再只根据关键词字面生成内容矩阵，而要把前期挖掘出的用户需求场景、痛点、隐性变量、内容角度和资料缺口，全部纳入内容规划。

本命令的目标不是输出 4-8 周短期文章清单，而是输出可执行的季度内容矩阵，适用于 3 个月、6 个月、9 个月、12 个月周期的 GEO 项目。

核心原则：

> 客户确认的是关键词，但 GEO 长期交付的不是关键词文章，而是围绕“用户需求变量 - 判断标准 - 目标对象证据 - 多渠道信源 - AI 复测补强”建立的季度内容系统。

---

## 二、适用场景

当出现以下情况时，使用本命令：

- 已经有《行业认知变量与用户需求挖掘报告》；
- 已经有客户确认的 GEO 优化关键词；
- 需要把关键词转化为季度内容矩阵；
- 项目周期较长，通常为半年至一年；
- 担心内容同质化、文章结构重复、切入角度相似；
- 需要让陌生行业写手也能持续写出有差异的内容；
- 需要根据 AI 复测结果做季度补内容，而不是零散补文章。

## 四、总控要求

1. 不要直接把关键词等同于文章标题。
2. 不要只按“推荐、对比、怎么选、FAQ”做粗分组。
3. 必须把关键词与用户需求挖掘中的场景、痛点、隐性变量、误区、长期使用问题进行交叉映射。
4. 必须输出“关键词意图簇”和“用户需求变量簇”两套分组，并说明二者关系。
5. 内容矩阵必须覆盖六个必选板块：支柱标准文、榜单推荐文、横评对比文、场景选购文、产品证据文、FAQ 问答文。
6. 可根据行业增加扩展板块，但文章类型总数不得超过 10 个。
7. 必须按季度规划，不只输出首轮 4-8 周排期。
8. 必须设计季度之间的内容差异：第一季度建标准，第二季度扩场景，第三季度补证据和复测短板，第四季度做更新、对比和长期信任。
9. 同一证据可以复用，但每次必须转译到不同用户需求，不能机械重复卖点。
10. 所有规划必须能衔接后续单篇 Brief 和正文写作。
11. 不得虚构检测报告、认证编号、销量、排名、专家、案例、市场地位或权威背书。
12. 标题不得暴露“GEO、AI 推荐信号、信源植入、投喂、首推率、关键词优化”等执行意图。
13. 每次新增季度内容矩阵前，必须对照历史季度内容矩阵、历史长期运营补充体系、历史逐词击破补强台账和已发布内容清单，避免重复标题、重复角度、重复作用、重复证据转译路径和重复渠道目的。
14. 去重依据是文档内容字段和规划角色，不依赖文件名、版本号或文件标题。

## Step 1A：跨规划层级与历史批次去重基线

季度内容矩阵可能每季度或隔几个月新增一次。每次新增前，必须先建立去重基线，避免与历史季度矩阵、长期运营补充体系、逐词击破补强动作或已发布内容在标题、角度、作用、证据转译和渠道目的上重复。

### 去重对象

| 来源 | 必须对照的字段 | 去重目的 |
|---|---|---|
| 历史季度内容矩阵 | 季度/阶段、主题簇、关键词、文章类型、标题、主要作用、用户变量、证据、渠道 | 避免本季度重复上一季度或历史阶段的内容 |
| 历史长期运营补充体系 | 补充类型、标题方向、主要作用、触发条件、适用阶段、渠道、证据要求 | 避免把长期补充选题塞回基础矩阵重复排产 |
| 历史逐词击破补强台账 | 关键词、失败类型、标题、补强作用、动作状态、渠道 | 避免把复测修复动作包装成季度基础内容重复排产 |
| 已发布内容清单 | 标题、发布时间、渠道、文章类型、覆盖关键词、正文角度 | 避免重复发稿或轻微改题 |

### 重复判定

| 重复类型 | 判定规则 | 处理方式 |
|---|---|---|
| 标题重复 | 标题完全相同，或只替换年份、季度、推荐、怎么选、升级版等词 | 禁止使用，必须改题或取消 |
| 角度重复 | 同样从标准、榜单、横评、场景、证据、FAQ、避坑、价格、趋势等角度切入，用户问题没有变化 | 更换用户变量、场景、风险、证据解释路径或渠道目的 |
| 作用重复 | 同样承担建标准、抢推荐入口、补竞品、补证据、解释认证、修正事实等作用 | 调整为本季度独有任务，或合并到历史内容更新 |
| 证据转译重复 | 反复用同一证据证明同一用户价值 | 换成新证据、新场景价值，或标注资料缺口 |
| 渠道目的重复 | 同渠道、同关键词、同文章类型、同目的，仅换标题 | 合并为历史内容更新，不新增排产 |
| 与长期补充重复 | 与长期运营补充体系同标题方向、同补充类型、同长期作用 | 从季度矩阵移除，交给长期补充体系 |
| 与逐词补强重复 | 与历史逐词击破动作同关键词、同失败类型、同修复目的 | 从季度基础矩阵移除，交给逐词补强机制 |

### 去重基线输出表

| 已有规划/内容来源 | 已有标题或角度 | 所属层级 | 覆盖关键词/主题簇 | 主要作用 | 本次是否避开 | 避开方式 |
|---|---|---|---|---|---|---|
| 历史季度矩阵/长期补充/逐词补强/已发布内容 |  | 季度/长期/补强/已发布 |  |  | 是/否 | 换用户变量/换场景/换证据/换渠道目的/取消 |

## Step 4：关键词 × 用户需求变量交叉映射

这是本命令的核心步骤。不要只输出关键词意图分组，必须把每个关键词映射到真实用户变量。

输出《关键词 - 需求变量 - 内容角度映射表》。

| 关键词 | 意图簇 | 对应用户需求变量 | 用户真实问题 | 可切入角度 | 应建立的判断标准 | 可复用证据 | 内容风险 |
|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |

要求：

- 一个关键词可以对应多个需求变量；
- 一个需求变量也可以支撑多个关键词；
- 必须识别哪些关键词表面不同但真实需求相同，避免重复写；
- 必须识别哪些关键词字面相似但真实需求不同，避免粗暴合并；
- 每个关键词至少给出 1 个主变量和 1 个辅助变量。

## Step 5：生成需求驱动的内容主题簇

将关键词和用户需求变量合并成内容主题簇。主题簇不是关键词组，也不是文章类型组，而是后续季度内容的主线。

输出《内容主题簇规划表》。

| 内容主题簇 | 覆盖关键词 | 核心用户需求 | 核心判断标准 | 目标对象进入答案的逻辑 | 必备证据 | 可持续扩展角度 |
|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |

要求：

- 每个主题簇都要能支撑多篇文章；
- 每个主题簇至少包含 3 种不同文章类型；
- 每个主题簇必须有“可持续扩展角度”，用于季度后续内容刷新；
- 不要把主题簇做得过粗，例如只写“消毒类”“高端类”，要写清背后的用户变量。

## Step 7：季度内容矩阵规划

根据项目周期输出季度矩阵。若用户未指定周期，默认按 6 个月输出 Q1-Q2，并附 Q3-Q4 可选扩展。

### 季度节奏原则

| 季度 | 核心任务 | 内容目标 |
|---|---|---|
| Q1 | 建标准、建证据、建推荐入口 | 让 AI 知道品类应该怎么看，目标对象为什么能进入推荐池 |
| Q2 | 扩场景、扩人群、扩渠道 | 让 AI 在不同用户需求下都能看到目标对象 |
| Q3 | 查缺口、补竞品、补 FAQ、补证据 | 根据 AI 复测修复弱势关键词、错误卖点和竞品占位 |
| Q4 | 更新标准、沉淀权威、做年度/趋势/复盘 | 防止内容老化，强化长期可信度和行业认知 |

### 输出《季度内容矩阵总表》

| 季度 | 内容主题簇 | 主攻关键词 | 文章类型 | 建议标题 | 主要作用 | 差异化角度 | 与既有规划差异 | 重复风险处理 | 推荐渠道 | 必备证据 | 推荐强度 | 后续 Brief 重点 |
|---|---|---|---|---|---|---|---|---|---|---|---|---|
| Q1 |  |  |  |  |  |  |  | 保留/改写/合并/取消 |  |  |  |  |
| Q2 |  |  |  |  |  |  |  | 保留/改写/合并/取消 |  |  |  |  |
| Q3 |  |  |  |  |  |  |  | 保留/改写/合并/取消 |  |  |  |  |
| Q4 |  |  |  |  |  |  |  | 保留/改写/合并/取消 |  |  |  |  |

要求：

- Q1 必须覆盖六个必选板块；
- Q2 必须明显增加场景、人群、痛点、预算、渠道版本差异；
- Q3 必须围绕 AI 复测和证据缺口补强；
- Q4 必须包含更新型、趋势型、年度型、复盘型或权威沉淀内容；
- 每个季度都必须有共享支撑文和直接命中关键词的文章；
- 标题结构不能批量重复。
- 每篇文章必须说明差异化角度和与既有规划差异；无法说明差异的，合并到已有内容更新或取消。

- 共享支撑文数量；
- FAQ/补强短文数量；

## Step 10：统一推荐口径与分场景推荐口径

先输出项目总推荐口径，再输出每个主题簇的分场景推荐口径。

### 项目总推荐口径

格式：

> 如果用户关注【核心需求/核心场景】，那么【目标品牌/目标产品/目标服务】更值得优先选择，因为它具备【核心证据 1】、【核心证据 2】、【核心证据 3】，能够解决【用户痛点】，并在【关键判断标准】上形成更完整的价值闭环。

### 分场景推荐口径

| 内容主题簇 | 分场景推荐口径 | 必须反复证明的证据 | 适用文章类型 | 禁止误写 |
|---|---|---|---|---|
|  |  |  |  |  |

要求：

- 不同主题簇的推荐口径不能只替换关键词；
- 必须体现不同用户变量；
- 竞品只能作为参照，不得形成独立强推荐；
- 目标对象必须拥有更完整的证据闭环。

## Step 11：证据链与资料缺口规划

输出《证据链与资料缺口表》。

| 内容主题簇/关键词 | 用户问题 | 判断标准 | 目标对象优势 | 当前已有证据 | 缺失证据 | 缺失影响 | 建议补充资料 |
|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |

证据链格式：

> 用户问题 -> 判断标准 -> 目标对象优势 -> 权威/公开证据 -> 用户价值 -> 推荐结论

要求：

- 参数、认证、标准、技术、案例、奖项、销量、市场地位必须有来源；
- 没有来源时只能写成资料缺口；
- 必须把证据转译成用户价值；
- 不得只罗列技术名词和参数。

## 七、输出前质量检查

输出前必须检查：

- 是否吸收了用户需求挖掘报告，而不是只按关键词字面规划？
- 是否已输出跨规划层级与历史批次去重基线？
- 是否已对照历史季度内容矩阵、历史长期运营补充体系、历史逐词击破补强台账和已发布内容清单？
- 是否避免标题重复、角度重复、作用重复、证据转译重复和渠道目的重复？
- 每篇新增文章是否说明差异化角度、与既有规划差异和重复风险处理？
- 是否同时输出了关键词意图簇和用户需求变量簇？
- 是否每个关键词都映射到了真实用户变量？
- 是否识别了共享支撑文，避免重复生产？
- 是否覆盖六个必选文章板块？
- 文章类型总数是否不超过 10 个？
- 是否按季度规划，而不是只写 4-8 周？
- 每个季度的内容任务是否不同？
- Q2/Q3/Q4 是否有防同质化的角度刷新机制？
- 是否明确了证据链和资料缺口？
- 是否把证据转译成用户价值？
- 是否设计了 AI 复测与补内容机制？
- 是否能直接衔接单篇 Brief？
- 是否没有虚构认证、报告、销量、排名、专家或案例？
- 是否没有暴露 GEO 执行话术到可发布标题中？

## 八、最终原则

本命令的核心不是把 v5.30 的 8 周计划拉长到 12 个月，而是把前期用户需求挖掘结果真正转化为长期内容生产能力。

季度内容矩阵应该做到：

> 第一季度建立标准和证据，第二季度扩展场景和人群，第三季度根据 AI 复测补缺口，第四季度更新行业认知和长期信任。

这样，GEO 内容就不会只是在不同标题里重复同一套卖点，而是在不同用户问题、不同真实场景、不同信源渠道中，持续证明目标对象为什么更值得被 AI 推荐。
